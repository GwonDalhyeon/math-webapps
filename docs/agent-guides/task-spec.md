# 작업지시서 작성 지침

이 문서는 Codex, Claude Code 또는 다른 구현 도구가 읽을 작업지시서를 만들 때 읽는다.

## 위치

공개 가능한 교육용 기획서와 작업지시서는 `tasks/`에 둔다.

파일명은 다음 형식을 권장한다.

```text
YYYY-MM-DD-topic-name.md
```

예:

```text
2026-06-10-sample-proportion-event-readiness.md
```

비공개 임시 지시서, 개인 메모, 실험용 프롬프트는 `tasks/`에 두지 않는다.

## 작업지시서의 역할

작업지시서는 구현자가 다음을 혼동하지 않게 해야 한다.

- 만들 웹앱의 수학적 목표
- 왜 웹앱이어야 하는지
- 학생의 핵심 조작과 아하포인트
- 예상 오개념과 이를 줄이는 장치
- 기본값과 조작값
- 화면에 반드시 보여야 할 상태
- 어떤 파일을 만들고 수정해야 하는지
- 어떤 기술을 쓰면 안 되는지
- 완료와 검증 기준

## 작성 기준

- 구현자가 바로 작업할 수 있을 만큼 구체적으로 쓴다.
- 사전 학습가치 검토를 통과할 수 있도록 웹앱이 필요한 이유와 아하포인트를 분명히 쓴다.
- 기본값은 학습지 계산값과 맞춘다.
- `n`, `p`, 반복 횟수처럼 헷갈리기 쉬운 기호는 역할을 분명히 쓴다.
- 수학 모델은 고등학생 수준 설명과 구현 계산식을 함께 둔다.
- 외부 라이브러리를 쓰지 않는 방향을 먼저 고려한다.
- CDN이 필요하면 이유와 오프라인 제약을 적는다.
- 실제 파일이 없는 링크를 만들지 말라고 명시한다.
- 웹앱에 맞는 학습지를 함께 만들 때는 Markdown/PDF를 웹앱 폴더에 두고, 웹앱 `index.html`의 제목/짧은 설명 아래에 다운로드 영역을 연결하라고 명시한다.

## 학습가치 기준 작성

작업지시서에는 구현 전에 검토자가 판단할 수 있는 기준을 포함한다.

좋은 예:

- 학생이 슬라이더로 표본 크기를 바꾸면 표본비율의 흔들림이 작아지는 장면이 즉시 보여야 한다.
- 아하포인트는 "한 번의 결과는 흔들리지만 표본 크기가 커질수록 분포 전체가 좁아진다"이다.
- 판서만으로는 반복 표집과 분포 형성 과정을 동시에 보여주기 어렵기 때문에 웹앱이 필요하다.
- 오개념 방지를 위해 표본 크기와 반복 횟수를 서로 다른 조작으로 분리해 라벨링한다.

피할 예:

- 확률을 쉽게 이해하게 한다.
- 그래프를 예쁘게 보여준다.
- 학생이 재미있게 조작한다.

## 수식

작업지시서 자체의 수식은 구현자가 이해하기 쉬우면 텍스트 표기와 LaTeX를 함께 사용할 수 있다.

다만 학습지 Markdown을 생성하도록 지시할 때는 반드시 다음 규칙을 포함한다.

```text
학생용 학습지와 예시답안이 있는 교사용 학습지를 함께 제작한다.
학습지 Markdown의 모든 수식은 LaTeX 형식으로 작성한다.
인라인 수식은 $...$, 별도 줄 수식은 $$...$$ 형식을 사용한다.
웹앱 폴더에 Markdown/PDF를 함께 두고 웹앱에서 다운로드할 수 있게 연결한다.
```

## 완료 기준 작성

완료 기준은 기능 단위로 확인 가능해야 한다.

좋은 예:

- `1회 실행`, `10회 실행`, `100회 실행`, `자동 실행`, `일시정지`, `초기화`가 안정적으로 동작한다.
- 자동 실행 중 중복 타이머가 생기지 않는다.
- `n`, `p`, `r`을 바꾸면 준비 인원, 그래프, 해석 문구가 즉시 갱신된다.
- 브라우저 콘솔 오류가 없다.
- 과목별 목록 페이지에 새 웹앱 링크가 추가되어 있다.
- 학습지 PDF/Markdown 다운로드 링크가 실제 파일을 가리킨다.

피할 예:

- 잘 만든다.
- 예쁘게 만든다.
- 오류가 없게 한다.

## 검증 방법 작성

검증 방법은 구현자가 따라 할 수 있는 순서형 체크리스트로 쓴다.

예:

```md
1. 웹앱을 브라우저에서 연다.
2. 기본값이 학습지와 일치하는지 확인한다.
3. `1회 실행`을 여러 번 눌러 결과가 달라지는지 확인한다.
4. 자동 실행을 켠 뒤 다시 눌러도 중복 실행되지 않는지 확인한다.
5. 개발자 도구 콘솔에 오류가 없는지 확인한다.
```
