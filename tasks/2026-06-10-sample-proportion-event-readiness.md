# 표본비율과 행사 준비 실험실 웹앱 작업지시서

웹앱 이름:
표본비율과 행사 준비 실험실

위치:
`probability-statistics/sample-proportion-event-readiness/index.html`

함께 만들 학습지:
`probability-statistics/sample-proportion-event-readiness/worksheet.md`

대상 학생:
고등학교 확률과 통계 과목에서 모비율, 표본비율, 이항분포, 표본비율의 분포를 처음 배우는 일반계 고등학교 학생

수업 상황:
교사는 "평균적으로 신청자의 70%가 참석한다면 70%만 준비해도 항상 충분할까?"라는 질문으로 수업을 시작한다. 웹앱은 교사가 조작하며 시연한다. 학생은 웹앱을 직접 조작하지 않고, 교사의 시연과 함께 배포된 학습지를 통해 표본비율이 모비율과 매번 일치하지 않는다는 점을 확인하고 적용한다.

이 앱이 보여줄 핵심은 다음이다. 학생이 가장 흔하게 갖는 오개념은 "평균만큼 준비하면 평균적으로 맞는다"는 생각이다. 그러나 `p`만큼만 준비하면(`r = 0`, `c = np`) 실제 참석자가 준비 인원을 넘는 일이 약 40% 확률로 발생한다. 기댓값이 14명이라는 것이 "14명 이하가 오는 행사가 더 많다"는 뜻이 아니기 때문이다. 이 앱은 그 확률(`P(X > c)`)을 계산·시각화하여 직관과 실제 확률 사이의 간격을 보여준다.

학습 목표:
- 모비율 `p`와 한 번의 표본에서 나타나는 표본비율 `X / n`을 구분한다.
- `p = 0.7`이라는 말이 매 행사마다 정확히 70%가 참석한다는 뜻이 아님을 이해한다.
- 신청자 수 `n`이 작을수록 표본비율의 흔들림이 크게 나타날 수 있음을 확인한다.
- 같은 조건의 행사를 여러 번 반복하면 표본비율들이 모여 분포를 만든다는 점을 관찰한다.
- 준비 비율을 `q = p + r`로 정했을 때 준비 부족이 발생할 확률을 이항분포로 해석한다.
- 준비를 많이 하면 부족 위험은 줄지만 남는 준비가 늘어나는 현실적 균형을 이해한다.

핵심 개념:
- 모비율
- 표본비율
- 이항분포
- 표본비율의 분포
- 반복 시행
- 준비 부족 확률
- 확률과 의사결정

## 핵심 상황

어떤 행사는 신청자 중 일부만 실제로 참석한다. 각 신청자가 실제로 참석할 확률이 `p`라고 알려져 있고, 한 행사 신청자 수가 `n`명이라고 하자.

이때 실제 참석자 수 `X`는 매 행사마다 달라진다.

```text
X ~ Binomial(n, p)
표본비율 = X / n
```

담당자가 신청자의 `q`비율만큼 준비하면 준비 인원은 다음과 같다.

```text
준비 비율 q = p + r
준비 인원 c = ceil(nq)
```

실제 참석자 수가 준비 인원보다 많으면 준비 부족 문제가 발생한다.

```text
문제 발생 조건: X > c
문제 발생 확률: P(X > c)
```

단, 화면에서는 학생이 이해하기 쉽도록 `ceil(nq)`로 준비 인원을 정한다. 예를 들어 `n = 20`, `p = 0.7`, `r = 0.1`이면 `q = 0.8`, 준비 인원은 16명이다.

기본 상태에서의 준비 부족 확률 참고값:

```text
n = 20, p = 0.7, r = 0   → c = 14, P(X > 14) ≈ 0.416 (약 10번 중 4번 부족)
n = 20, p = 0.7, r = 0.1 → c = 16, P(X > 16) ≈ 0.107 (약 10번 중 1번 부족)
n = 100, p = 0.7, r = 0.1 → c = 80, P(X > 80) ≈ 0.009 (약 100번 중 1번 부족)
```

`r = 0`일 때 준비 부족 확률이 약 40%라는 것이 수업의 핵심 충격이다. 이 값이 앱을 열자마자 눈에 잘 띄어야 한다.

## 기본값

웹앱을 처음 열었을 때 학습지와 바로 연결되도록 다음 값을 기본값으로 둔다.

```text
한 행사 신청자 수 n = 20
실제 참석 확률 p = 0.7
여유 준비 비율 r = 0.0
준비 비율 q = p + r = 0.7
준비 인원 c = 14명
반복 행사 횟수 m = 100
```

빠른 비교 버튼 또는 프리셋을 제공한다.

- `p만큼 준비`: `n = 20`, `p = 0.7`, `r = 0.0`, `q = 0.7`, `c = 14`
- `10%p 더 준비`: `n = 20`, `p = 0.7`, `r = 0.1`, `q = 0.8`, `c = 16`
- `큰 행사 비교`: `n = 100`, `p = 0.7`, `r = 0.1`, `q = 0.8`, `c = 80`

`r`은 퍼센트포인트 단위의 여유 비율이다. 예를 들어 `p = 0.7`, `r = 0.1`이면 준비 비율은 0.8이다.

## 사용자 조작

### 슬라이더와 입력값

- 신청자 수 `n`
  - 범위: 10부터 200
  - 기본값: 20
  - 정수 단위
- 실제 참석 확률 `p`
  - 범위: 0.1부터 0.95
  - 기본값: 0.7
  - 0.05 단위
- 여유 준비 비율 `r`
  - 범위: 0부터 0.3
  - 기본값: 0
  - 0.01 또는 0.05 단위
  - `p + r`이 1을 넘으면 준비 비율은 1로 제한한다.
- 반복 행사 횟수 `m`
  - 선택 버튼: 10회, 50회, 100회, 500회
  - 기본값: 100회

### 실행 버튼

- `1회 실행`
  - 한 번의 행사를 시뮬레이션하고 실제 참석자 수와 표본비율을 보여준다.
- `10회 실행`
  - 같은 조건으로 10번의 행사를 추가로 시뮬레이션한다.
- `100회 실행`
  - 같은 조건으로 100번의 행사를 추가로 시뮬레이션한다.
- `자동 실행`
  - 일정 간격으로 행사를 반복 실행한다.
- `일시정지`
  - 자동 실행을 멈춘다.
- `초기화`
  - 누적 기록과 그래프를 지운다.

자동 실행 중에는 중복 타이머가 생기지 않도록 한다. 슬라이더 값이 바뀌면 자동 실행을 멈추고 누적 기록을 초기화한다.

## 화면 구성

화면은 다음 흐름으로 구성한다.

```text
상황 설명 -> 조작 패널 -> 한 행사 결과 -> 반복 행사 결과 -> 분포와 해석
```

### 1. 상황 설명

상단에는 학생의 직관을 먼저 건드리는 질문으로 시작한다.

```text
"평균적으로 70%가 참석하니, 70%만 준비하면 충분하다."
이 판단, 얼마나 자주 틀릴까요?
```

그 아래에 개념 정리 문장을 둔다.

```text
모비율 p는 장기적으로 기대되는 참석 비율입니다.
하지만 한 행사에서 실제로 나타나는 표본비율 X/n은 매번 달라지고,
p만큼만 준비하면 약 40%의 행사에서 준비가 부족합니다.
```

### 2. 조작 패널

교사가 수업 중 바로 값을 바꿀 수 있도록 큰 슬라이더와 현재 값을 함께 보여준다.

반드시 표시할 값:

- 신청자 수 `n`
- 실제 참석 확률 `p`
- 여유 준비 비율 `r`
- 준비 비율 `q = p + r`
- 준비 인원 `c`
- **준비 부족 확률 `P(X > c)`** — 슬라이더와 나란히 크게 표시한다. 조작 패널의 핵심 수치이며 `n`, `p`, `r`이 바뀔 때마다 즉시 갱신된다. 기본 상태(`r = 0`)에서 약 40%임을 한눈에 알 수 있어야 한다.

### 3. 한 행사 결과

가장 최근 실행한 행사 결과를 크게 보여준다.

- 실제 참석자 수 `X`
- 표본비율 `X / n`
- 준비 인원 `c`
- 결과 상태
  - 충분: `X <= c`
  - 부족: `X > c`

가능하면 신청자 `n`명을 작은 점 배열로 나타낸다.

- 참석한 사람: 진한 색 점
- 불참한 사람: 연한 회색 점
- 준비 인원 기준선 또는 표식

### 4. 반복 행사 결과

누적 시뮬레이션 결과를 보여준다.

- 누적 행사 수
- 문제 발생 횟수
- 문제 발생 비율
- 평균 실제 참석자 수
- 평균 표본비율

문제 발생 비율은 다음처럼 표현한다.

```text
100번 중 8번 준비 부족
문제 발생 비율 8.0%
```

### 5. 분포 시각화

핵심 그래프는 이항분포 막대그래프이다.

- 가로축: 실제 참석자 수 `X`
- 세로축: 확률 또는 누적 실험 횟수
- 준비 인원 `c`를 세로 기준선으로 표시
- `X > c`인 막대를 강조색(예: 붉은 계열)으로 칠하여 준비 부족 영역을 명확히 구분한다
- `X ≤ c`인 막대는 다른 색(예: 파란 계열)으로 표시하여 대비를 분명히 한다
- 그래프 상단 또는 부족 영역 위에 **`P(X > c) = 0.416`** 처럼 준비 부족 확률 수치를 크고 명확하게 표시한다. 이 숫자가 그래프의 핵심 메시지이다.
- 현재까지 실제로 관찰된 행사 결과를 점 또는 막대 위 표시로 겹쳐 보여줄 수 있다.

그래프 전환 옵션을 제공하면 좋다.

- `이론분포`: `X ~ Binomial(n, p)`의 확률 막대
- `실험분포`: 반복 실행으로 관찰된 실제 참석자 수의 도수 막대

필수는 아니지만, 가능하면 두 분포를 같은 축에서 비교할 수 있게 한다.

## 해석 문구

화면 오른쪽 또는 하단에 현재 조건에 맞는 해석 문구를 자동으로 갱신한다.

예시:

```text
p = 0.7이므로 평균적으로는 20명 중 14명이 참석합니다. 하지만 실제 참석자 수는 매번 달라집니다.
```

```text
14명분을 준비하는 것은 '평균만큼 준비하는 것'입니다.
그런데 15명 이상 오는 행사의 확률은 약 41.6%입니다.
평균만큼 준비하면 10번 중 4번은 부족합니다.
```

```text
16명분을 준비하면 부족 확률은 약 10.7%로 줄어듭니다.
하지만 참석자가 적은 행사에서는 남는 준비가 생깁니다.
```

```text
신청자 수가 커질수록 표본비율 X/n은 p 근처에 더 모이는 경향이 있습니다.
n = 100으로 늘려도 70%만 준비하면 부족 확률은 여전히 약 40%입니다.
```

r을 0에서 올릴 때마다 준비 부족 확률이 얼마나 줄어드는지 비교하는 문구도 제공한다.

```text
r = 0   → P(X > c) ≈ 41.6%
r = 0.1 → P(X > c) ≈ 10.7%
r = 0.2 → P(X > c) ≈ 1.0%
```

## 수학 모델 또는 계산식

### 이항분포 확률

```text
P(X = k) = C(n, k) p^k (1 - p)^(n - k)
```

직접 조합값을 큰 수로 계산하면 오버플로가 날 수 있으므로 재귀 비율로 확률 배열을 만든다.

```text
P(X = 0) = (1 - p)^n
P(X = k + 1) = P(X = k) * (n - k) / (k + 1) * p / (1 - p)
```

`p`가 1에 매우 가까운 경우를 고려해 `p = 1`은 허용하지 않는다. 슬라이더 상한은 0.95로 둔다.

### 준비 부족 확률

```text
c = ceil(nq)
P(준비 부족) = P(X > c)
```

화면에서는 `c`명을 준비했을 때 `c + 1`명 이상 참석하면 문제가 생긴다고 설명한다.

### 표본비율의 평균과 표준편차

필요하면 보조 설명으로 표시한다.

```text
E(X / n) = p
SD(X / n) = sqrt(p(1 - p) / n)
```

이 식은 웹앱의 중심 내용은 아니지만, `n`이 커질수록 표본비율의 흔들림이 줄어드는 이유를 설명할 때 사용한다.

## 학습지 연계

학습지는 배운 내용을 적용하는 활동지다. 학생은 웹앱을 직접 조작하지 않는다. 교사가 웹앱을 시연하는 동안 학생은 학습지에 주어진 데이터와 확률값을 바탕으로 계산하고 판단한다. 웹앱이 없어도 학습지는 완성할 수 있어야 한다.

학습지 흐름:

```text
모비율·표본비율 구분 -> 주어진 데이터로 표본비율 계산 ->
준비 인원별 부족 확률 비교 -> 의사결정 -> n의 역할 -> 개념 정리 + 열린 질문
```

학습지 기본값은 웹앱 기본값과 맞춘다.

```text
n = 20
p = 0.7
```

학생이 직접 계산할 값은 다음처럼 단순하게 유지한다.

- `20 × 0.7 = 14` (기댓값, 전략 A 준비 인원)
- `20 × 0.8 = 16` (전략 B 준비 인원)
- `100 × 0.7 = 70` (n = 100 기댓값)
- `100 × 0.8 = 80` (n = 100 전략 B 준비 인원)
- 표본비율 계산: `16/20 = 0.8`, `12/20 = 0.6`, `14/20 = 0.7`

확률값은 학생이 계산하지 않고 학습지에 주어진 값을 읽고 해석한다.

```text
P(X > 14) ≈ 0.416  (c = 14, n = 20)
P(X > 16) ≈ 0.107  (c = 16, n = 20)
P(X > 80) ≈ 0.009  (c = 80, n = 100)
```

학습지는 모비율 추정을 별도 절로 다루지 않고, 개념 정리 마지막에 열린 질문 한 문장으로 마무리한다.

```text
"실제로는 p를 모르는 경우가 더 많습니다. 관찰한 표본비율로 p를 어떻게 추정할 수 있을까요?"
```

## 완료 기준

- 웹앱이 GitHub Pages에서 정적 HTML 하나로 바로 실행된다.
- 기본값이 학습지와 일치한다.
- `1회 실행`, `10회 실행`, `100회 실행`, `자동 실행`, `일시정지`, `초기화`가 안정적으로 동작한다.
- 자동 실행 중 중복 타이머가 생기지 않는다.
- `n`, `p`, `r`을 바꾸면 준비 인원, 그래프, 해석 문구가 즉시 갱신된다.
- 한 행사 결과와 누적 결과가 서로 구분되어 표시된다.
- 이항분포 그래프에서 준비 부족 영역 `X > c`가 명확히 강조된다.
- 그래프에 `P(X > c)` 수치가 크게 표시된다.
- 기본 상태(`n = 20`, `p = 0.7`, `r = 0`)에서 조작 패널과 그래프 모두에서 준비 부족 확률 약 41.6%가 눈에 잘 띈다.
- `r`을 0에서 올릴 때 준비 부족 확률이 즉시 갱신되어 감소를 확인할 수 있다.
- 반복 실험에서 문제 발생 횟수와 문제 발생 비율이 표시된다.
- `p`와 표본비율 `X / n`의 차이를 설명하는 문구가 있다.
- 브라우저 콘솔 오류가 없다.
- 모바일 화면에서도 주요 조작과 그래프가 깨지지 않는다.
- `probability-statistics/index.html`에 새 웹앱 링크가 추가되어 있다.

## 검증 방법

1. `probability-statistics/sample-proportion-event-readiness/index.html`을 브라우저에서 연다.
2. 기본값이 `n = 20`, `p = 0.7`, `r = 0`, 준비 인원 14명으로 표시되는지 확인한다.
3. `1회 실행`을 여러 번 눌러 실제 참석자 수와 표본비율이 매번 달라지는지 확인한다.
4. `100회 실행`을 눌러 누적 행사 수, 문제 발생 횟수, 문제 발생 비율이 갱신되는지 확인한다.
5. 여유 준비 비율을 `r = 0.1`로 바꾸어 준비 인원이 16명으로 바뀌는지 확인한다.
6. 같은 조건에서 문제 발생 영역이 줄어드는지 그래프와 수치로 확인한다.
7. 신청자 수를 `n = 100`으로 바꾸어 표본비율 분포가 더 좁아지는지 확인한다.
8. 자동 실행을 켠 뒤 다시 자동 실행 버튼을 눌러도 실행 속도가 중복으로 빨라지지 않는지 확인한다.
9. 초기화를 누르면 누적 기록과 실험분포가 지워지는지 확인한다.
10. 개발자 도구 콘솔에 오류가 없는지 확인한다.
