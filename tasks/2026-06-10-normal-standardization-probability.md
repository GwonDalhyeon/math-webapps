# 정규분포 확률의 표준화 웹앱 작업지시서

웹앱 이름:
정규분포와 표준정규분포: 같은 확률 보기

권장 위치:
`probability-statistics/normal-standardization-probability/index.html`

대상 학생:
고등학교 확률과 통계에서 정규분포, 표준정규분포, 표준화, 정규분포의 확률 계산을 배우는 일반계 고등학교 학생

수업 상황:
표준점수와 T점수 변환을 배운 뒤 바로 이어지는 차시가 아니라, 중간에 1-2시간 다른 내용이 설명된 뒤 정규분포의 확률 계산을 다루는 상황을 가정한다. 따라서 기존 `standard-score-t-score` 앱을 확장하지 말고 별도 웹앱으로 만든다. 이 앱은 T점수 변환이 아니라, 정규분포 `X ~ N(m, sigma^2)`에서 구간 `a <= X <= b`의 확률을 표준정규분포 `Z ~ N(0,1)`의 구간 확률로 바꾸어 보는 데 집중한다.

핵심 질문:

```text
정규분포에서 a 이상 b 이하일 확률은 표준정규분포에서 어디부터 어디까지의 확률일까?
```

핵심 메시지:

```text
정규분포의 값 a, b 자체보다 중요한 것은 각각이 평균에서 표준편차 몇 개만큼 떨어져 있는가이다.
a = m + z_a sigma, b = m + z_b sigma 이면
P(a <= X <= b) = P(z_a <= Z <= z_b) 이다.
```

조작 원칙:

```text
이 웹앱의 기준 상태는 a, b 자체가 아니라 z_a, z_b이다.
평균 m 또는 표준편차 sigma를 바꾸면 z_a, z_b는 그대로 두고
a = m + z_a sigma, b = m + z_b sigma를 다시 계산한다.
```

화면 조작의 중심:

```text
학생이 직접 조작하는 값은 m, sigma, z_a, z_b이다.
a, b는 직접 입력하지 않고 계산 결과로 표시한다.
그래프의 경계선은 위쪽 정규분포 그래프와 아래쪽 표준정규분포 그래프 모두에서 드래그할 수 있다.
```

---

## 학습 목표

- 정규분포 `X ~ N(m, sigma^2)`의 구간 `a <= X <= b`를 표준화할 수 있다.
- `z_a = (a-m)/sigma`, `z_b = (b-m)/sigma`의 의미를 평균에서 표준편차 몇 칸 떨어진 위치로 해석할 수 있다.
- 두 그래프에서 색칠된 영역이 서로 같은 확률을 나타낸다는 것을 이해한다.
- 평균 `m`과 표준편차 `sigma`가 바뀌어도 같은 `z` 구간이면 확률이 같다는 것을 시각적으로 확인한다.
- 정규분포 확률 계산에서 표준정규분포표 또는 계산 도구를 사용하는 이유를 설명할 수 있다.

---

## 기본 예시 데이터

기본 상태:

```text
m = 70
sigma = 10
a = 60
b = 90
```

이에 대응하는 표준화 값:

```text
z_a = (60 - 70) / 10 = -1
z_b = (90 - 70) / 10 = 2
```

화면에 보여줄 핵심 등식:

```text
P(60 <= X <= 90) = P(-1 <= Z <= 2)
```

해석 문장:

```text
60은 평균 70에서 표준편차 1개만큼 왼쪽에 있고, 90은 평균 70에서 표준편차 2개만큼 오른쪽에 있습니다.
따라서 원래 정규분포의 60부터 90까지의 확률은 표준정규분포의 -1부터 2까지의 확률과 같습니다.
```

## 화면 구성

화면은 한 페이지 안에서 다음 흐름을 가진다.

```text
짧은 설명 -> 조작 패널 -> 두 그래프 비교 -> 확률 등식과 해석
```

### 1. 상단 설명

제목:

```text
정규분포와 표준정규분포: 같은 확률 보기
```

짧은 설명:

```text
정규분포에서 구간의 확률을 구할 때는 각 경계값이 평균에서 표준편차 몇 개만큼 떨어져 있는지로 바꾸어 봅니다.
```

너무 긴 이론 설명을 넣지 않는다. 핵심은 그래프와 조작으로 보이게 한다.

### 2. 조작 패널

필수 입력:

- 평균 `m`
- 표준편차 `sigma`
- 왼쪽 표준화 경계 `z_a`
- 오른쪽 표준화 경계 `z_b`

입력 방식:

- `m`, `sigma`, `z_a`, `z_b`는 슬라이더와 숫자 입력을 함께 제공한다.
- `a`, `b`는 직접 입력하지 않고 `a = m + z_a sigma`, `b = m + z_b sigma`의 계산 결과로 표시한다.
- 위쪽 정규분포 그래프의 수직선 `x=a`, `x=b`를 마우스 또는 터치로 드래그할 수 있어야 한다.
- 아래쪽 표준정규분포 그래프의 수직선 `z=z_a`, `z=z_b`도 마우스 또는 터치로 드래그할 수 있어야 한다.
- 수직선을 움직이면 슬라이더와 숫자 입력이 함께 갱신된다.
- 슬라이더 또는 숫자 입력을 움직이면 그래프의 수직선도 함께 갱신된다.
- 위쪽 그래프에서 `x=a`, `x=b`를 드래그하면 그에 맞게 `z_a`, `z_b`가 다시 계산된다.
- 아래쪽 그래프에서 `z=z_a`, `z=z_b`를 드래그하면 `z_a`, `z_b`가 바뀌고 `a`, `b`가 다시 계산된다.
- `m` 또는 `sigma`를 조작하면 `z_a`, `z_b`는 그대로 유지하고 `a`, `b`를 다시 계산한다.

기본 범위 예시:

```text
m: 40부터 100
sigma: 2부터 25
z_a, z_b: -4부터 4
a, b: 현재 m, sigma, z_a, z_b로부터 자동 계산
```

주의:

- `sigma`는 반드시 0보다 커야 한다.
- `z_a > z_b`가 되지 않도록 한다. 경계선이 서로 넘어가지 않는 방식을 사용한다.
  - `z_a` 슬라이더의 최댓값을 항상 `z_b - 0.1`로 제한한다.
  - `z_b` 슬라이더의 최솟값을 항상 `z_a + 0.1`로 제한한다.
  - 드래그 중에도 같은 방식으로 제한한다.
  - 두 값을 교환하는 방식은 사용하지 않는다.

권장 버튼:

- `기본 예시`: `m=70`, `sigma=10`, `z_a=-1`, `z_b=2`로 완전 초기화한다. m, sigma까지 함께 초기화해야 수업 시연에서 기준점이 명확해진다.
- `대칭 구간`: 현재 m, sigma는 유지하고 `z_a=-1`, `z_b=1`로 설정한다.
- `두 표준편차`: 현재 m, sigma는 유지하고 `z_a=-2`, `z_b=2`로 설정한다.

단, 버튼이 너무 많아지면 `기본 예시`, `대칭 구간` 정도만 둔다. 이 앱은 `a <= X <= b`의 유한 구간 확률을 다루므로 왼쪽 꼬리, 오른쪽 꼬리 버튼은 넣지 않는다.

### 3. 원래 정규분포 그래프

그래프 제목:

```text
X ~ N(m, sigma^2)
```

시각화 요소:

- 정규분포 밀도곡선
- x축
- 평균 `m` 기준선
- `x = a`, `x = b` 수직선
- `a <= X <= b`에 해당하는 곡선 아래 영역 색칠
- x축 눈금에 실제 값과 표준편차 기준을 함께 표시

x축 눈금 표시 예시:

```text
40       50       60       70       80       90       100
m-3sigma m-2sigma m-sigma m        m+sigma  m+2sigma m+3sigma
```

화면에서는 다음처럼 실제 값과 표준편차 표현을 함께 보여준다.

```text
60
m - sigma
```

```text
90
m + 2sigma
```

경계선 라벨:

```text
x = a = 60 = m - 1sigma
x = b = 90 = m + 2sigma
```

학생용 화면에서는 `m + -1sigma`처럼 어색한 표현이 나오지 않게 한다.

- `z = -1`이면 `m - 1sigma`
- `z = 0`이면 `m`
- `z = 2`이면 `m + 2sigma`

처럼 부호를 자연스럽게 표시한다.

### 4. 표준정규분포 그래프

그래프 제목:

```text
Z ~ N(0, 1)
```

시각화 요소:

- 표준정규분포 밀도곡선
- z축
- 평균 `0` 기준선
- `z = z_a`, `z = z_b` 수직선
- `z_a <= Z <= z_b`에 해당하는 곡선 아래 영역 색칠
- 위쪽 정규분포 그래프와 같은 색의 영역을 사용한다.

축 눈금:

```text
-3, -2, -1, 0, 1, 2, 3
```

경계선 라벨:

```text
z_a = (a - m) / sigma = -1
z_b = (b - m) / sigma = 2
```

두 그래프의 수직선은 시각적으로 대응되게 한다.

- 위 그래프의 `x=a`와 아래 그래프의 `z=z_a`를 같은 색 또는 같은 라벨로 표시한다.
- 위 그래프의 `x=b`와 아래 그래프의 `z=z_b`를 같은 색 또는 같은 라벨로 표시한다.
- 색만으로 구분하지 않고 `a`, `b`, `z_a`, `z_b` 텍스트를 함께 표시한다.

### 5. 확률 등식과 현재 상태

두 그래프 사이의 중앙 띠에 현재 등식을 크게 표시한다. 이 위치는 위쪽 정규분포와 아래쪽 표준정규분포를 연결하는 다리 역할을 하며, 곡선이나 색칠 영역을 가리지 않는다.

예:

```text
P(60 <= X <= 90) = P(-1 <= Z <= 2) = 0.8186
```

교과서 형식의 LaTeX 예:

```text
\[
P(60 \le X \le 90)=P(-1 \le Z \le 2)\approx 0.8186
\]
```

확률 표시:

- 두 그래프 사이 중앙 띠에 확률 등식을 크게 표시한다.
- 각 그래프 안쪽 오른쪽 위에는 작은 확률 배지를 둘 수 있다.
- 확률 배지는 곡선, 색칠 영역, 축 눈금을 가리지 않는 위치에 둔다.
- `확률 표시` 토글을 둔다.
- 기본값은 확률 표시 켜짐이다.
- 토글을 끄면 중앙 등식과 그래프 안의 확률 배지를 숨기고, 그래프와 경계선은 그대로 둔다.

함께 보여줄 해석:

```text
a = 60은 m - 1sigma, b = 90은 m + 2sigma입니다.
두 그래프의 색칠된 영역은 축의 단위만 다를 뿐 같은 확률입니다.
```

확률값 표시:

- 소수 넷째 자리까지 표시한다.
- 예: `0.8186`
- 필요하면 백분율도 함께 표시한다.
- 예: `약 81.86%`

## 사용자 조작

필수 조작:

- 평균 `m` 슬라이더와 숫자 입력
- 표준편차 `sigma` 슬라이더와 숫자 입력
- 왼쪽 표준화 경계 `z_a` 슬라이더와 숫자 입력
- 오른쪽 표준화 경계 `z_b` 슬라이더와 숫자 입력
- 원래 정규분포 그래프에서 `x=a`, `x=b` 수직선을 드래그
- 표준정규분포 그래프에서 `z=z_a`, `z=z_b` 수직선을 드래그
- 확률 표시 켜기/끄기
- 기본 예시로 되돌리기

드래그 동작:

- `x=a` 수직선 위에 마우스를 올리면 커서가 좌우 이동 형태로 바뀐다.
- `z=z_a` 수직선 위에 마우스를 올려도 커서가 좌우 이동 형태로 바뀐다.
- 수직선 근처에 잡기 쉬운 원형 핸들을 둔다.
- 모바일 터치에서도 핸들을 움직일 수 있게 pointer events를 사용한다.
- 드래그 중에는 확률값과 등식이 즉시 갱신된다.
- 선이 그래프 밖으로 나가거나 다른 선을 넘어가지 않게 제한한다.

권장 추가 조작:

- `대칭 구간`: `a = m - sigma`, `b = m + sigma`
- `두 표준편차`: `a = m - 2sigma`, `b = m + 2sigma`

사용하지 말 것:

- T점수 변환
- 등급, 백분위, 석차 계산
- 여러 학생 점수 비교
- 표준점수와 T점수 앱의 기존 단계 변환 화면
- 왼쪽 꼬리, 오른쪽 꼬리처럼 한쪽 끝이 무한대로 이어지는 확률 모드
- 서버 기능, API 호출, 빌드 과정

## 시각화 세부 기준

그래프 구현:

- SVG를 우선 사용한다.
- 두 그래프는 하나의 SVG 안에 위/아래 구역으로 배치한다. 하나의 SVG를 사용하면 위아래 그래프의 x좌표가 공유되어 경계선 정렬과 연결선 구현이 단순해진다.
- 외부 그래프 라이브러리는 사용하지 않는다.
- 한 파일 `index.html` 안에 HTML, CSS, JavaScript를 작성한다.

그래프 배치:

- 두 그래프는 위아래로 배치한다.
- 위쪽: 원래 정규분포 `\(X \sim N(m,\sigma^2)\)`
- 아래쪽: 표준정규분포 `\(Z \sim N(0,1)\)`
- 위쪽 그래프의 `x=a`, `x=b` 수직선과 아래쪽 그래프의 `z=z_a`, `z=z_b` 수직선은 같은 화면 x좌표에 놓이게 한다.
- 두 그래프 사이의 여백에서도 같은 색의 얇은 연결선을 이어서, 경계선이 위아래 그래프를 관통하는 하나의 끊김없는 직선처럼 보이게 한다.
- 실제 수학 좌표는 다르지만 화면에서는 `a`와 `z_a`, `b`와 `z_b`가 같은 표준화 위치에 있다는 점을 보여주기 위해 정렬한다.

그래프 범위:

- 원래 정규분포 그래프는 기본적으로 `m - 4sigma`부터 `m + 4sigma`까지 표시한다.
- `z_a`, `z_b`가 `-4`부터 `4`까지 움직이도록 제한하면 `a`, `b`는 자동으로 이 범위 안에 놓인다.
- 표준정규분포 그래프는 기본적으로 `-4`부터 `4`까지 표시한다.

영역 색칠:

- `a <= X <= b`와 `z_a <= Z <= z_b` 영역은 같은 색으로 칠한다.
- 색은 과하게 진하지 않게 하되 프로젝터에서 분명히 보이게 한다.
- 곡선, 축, 수직선이 색칠 영역 위에서도 잘 보여야 한다.

축 라벨:

- 원래 정규분포 그래프는 실제 값과 `m +/- n sigma`를 함께 표시한다.
- 표준정규분포 그래프는 `z`값을 중심으로 표시한다.
- 평균 위치는 두 그래프 모두 굵게 표시한다.

수업 사용성:

- 그래프 위 텍스트가 서로 겹치지 않도록 라벨 위치를 조정한다.

- 교실 프로젝터에서 보이도록 선, 글씨, 핸들을 충분히 크게 만든다.

## 상태/결과 표시

항상 표시할 값:

```text
m
sigma
a
b
z_a = (a-m)/sigma
z_b = (b-m)/sigma
P(a <= X <= b)
P(z_a <= Z <= z_b)
```

사용자 조작 패널에서 직접 입력하는 값:

```text
m
sigma
z_a
z_b
```

계산 결과로 표시하는 값:

```text
a = m + z_a sigma
b = m + z_b sigma
```

핵심 등식:

```text
P(a <= X <= b) = P(z_a <= Z <= z_b)
```

예시 표시:

```text
P(60 <= X <= 90) = P(-1 <= Z <= 2) = 0.8186
```

해석 문장 템플릿:

```text
a는 평균에서 표준편차 ___개만큼 왼쪽/오른쪽에 있습니다.
b는 평균에서 표준편차 ___개만큼 왼쪽/오른쪽에 있습니다.
그래서 원래 정규분포의 이 구간은 표준정규분포의 ___부터 ___까지로 옮겨집니다.
```

## 수학 모델 또는 계산식

정규분포:

```text
X ~ N(m, sigma^2)
```

표준화:

```text
Z = (X - m) / sigma
z_a = (a - m) / sigma
z_b = (b - m) / sigma
a = m + z_a sigma
b = m + z_b sigma
```

확률 대응:

```text
P(a <= X <= b) = P(z_a <= Z <= z_b)
```

정규분포 밀도함수:

```text
f_X(x) = 1 / (sigma * sqrt(2pi)) * exp(-1/2 * ((x-m)/sigma)^2)
```

표준정규분포 밀도함수:

```text
phi(z) = 1 / sqrt(2pi) * exp(-z^2 / 2)
```

확률 계산:

```text
P(z_a <= Z <= z_b) = Phi(z_b) - Phi(z_a)
```

JavaScript에서는 표준정규분포 누적분포함수 `Phi(z)`를 근사식으로 구현한다. 별도 서버나 API를 사용하지 않는다.

권장 구현 방식:

```text
Phi(z) = 0.5 * (1 + erf(z / sqrt(2)))

erf(x) 근사 (Abramowitz & Stegun 7.1.26):
t = 1 / (1 + 0.3275911 * |x|)
erf(x) ≈ 1 - (a1*t + a2*t^2 + a3*t^3 + a4*t^4 + a5*t^5) * exp(-x^2)
a1 = 0.254829592, a2 = -0.284496736, a3 = 1.421413741
a4 = -1.453152027, a5 = 1.061405429
|error| < 1.5e-7 → 완료 기준 0.0002 이내 충분히 만족
```

주의:

- 고등학교 수업에서는 적분 계산 자체보다 구간이 표준화되는 의미를 우선한다.
- 확률값은 근사값임을 작은 문구로 표시한다.

## 완료 기준

- 별도 폴더 `probability-statistics/normal-standardization-probability/`에 웹앱이 생성되어 있다.
- 기본 상태에서 `m=70`, `sigma=10`, `a=60`, `b=90`이 표시된다.
- 원래 정규분포 그래프에 `x=a`, `x=b` 수직선과 `a <= X <= b` 색칠 영역이 보인다.
- 표준정규분포 그래프에 대응하는 `z_a=-1`, `z_b=2` 수직선과 같은 색칠 영역이 보인다.
- 원래 그래프의 x축에는 실제 값과 `\(m \pm n\sigma\)` 표현이 함께 표시된다.
- `z_a`, `z_b` 슬라이더를 움직이면 두 그래프와 확률값이 즉시 갱신된다.
- 위쪽 그래프의 `x=a`, `x=b` 수직선을 드래그하면 슬라이더, 숫자 입력, 표준화 값, 확률값이 즉시 갱신된다.
- 아래쪽 그래프의 `z=z_a`, `z=z_b` 수직선을 드래그해도 슬라이더, 숫자 입력, 원래 값 `a`, `b`, 확률값이 즉시 갱신된다.
- 위쪽과 아래쪽의 대응 경계선은 같은 화면 x좌표에 정렬되어 하나의 끊김없는 직선처럼 보인다.
- `z_a`와 `z_b`가 서로 넘어가지 않도록 안정적으로 제한된다.
- 평균 `m` 또는 표준편차 `sigma`를 바꾸면 `z_a`, `z_b`는 유지되고 `a`, `b`가 `a=m+z_a sigma`, `b=m+z_b sigma`로 다시 계산된다.
- `a`, `b`는 직접 입력하지 않고 계산 결과로만 표시된다.
- 확률 등식 `P(a <= X <= b) = P(z_a <= Z <= z_b)`이 현재 숫자와 함께 표시된다.
- 확률 표시는 기본값이 켜짐이며, 토글로 켜고 끌 수 있다.
- T점수 변환 내용은 포함하지 않는다.
- 왼쪽 꼬리, 오른쪽 꼬리 확률 모드는 포함하지 않는다.
- 검증 예시의 확률값은 표준값과 `0.0002` 이내이면 통과로 본다.
- 브라우저 콘솔 오류가 없다.
- 모바일 화면에서도 그래프와 조작 패널이 겹치지 않는다.
- `probability-statistics/index.html`에 실제 파일이 있는 링크만 추가되어 있다.

- 드래그 중 확률값과 등식이 즉시 갱신되는지 확인한다.

- 모바일 폭에서 조작 패널과 그래프가 깨지지 않는지 확인한다.

4. 수학 계산 확인

기본 예시:

```text
z_a = (60 - 70) / 10 = -1
z_b = (90 - 70) / 10 = 2
P(-1 <= Z <= 2) ≈ 0.8186
```

대칭 예시:

```text
a = m - sigma
b = m + sigma
z_a = -1
z_b = 1
P(-1 <= Z <= 1) ≈ 0.6827
```

두 표준편차 예시:

```text
a = m - 2sigma
b = m + 2sigma
z_a = -2
z_b = 2
P(-2 <= Z <= 2) ≈ 0.9545
```

평균/표준편차 변경 예시:

```text
처음: m = 70, sigma = 10, z_a = -1, z_b = 2
따라서 a = 60, b = 90

m = 80으로 바꾸면 z_a = -1, z_b = 2는 유지된다.
따라서 a = 70, b = 100으로 바뀐다.
확률 P(-1 <= Z <= 2)는 그대로 유지된다.

sigma = 5로 바꾸면 z_a = -1, z_b = 2는 유지된다.
따라서 a = 75, b = 90으로 바뀐다.
확률 P(-1 <= Z <= 2)는 그대로 유지된다.
```
