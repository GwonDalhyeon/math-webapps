# 구분구적법과 정적분: 구간 길이와 표본점 표현 비교 웹앱 작업지시서

웹앱 이름:
구분구적법과 정적분: x를 어떻게 잡느냐에 따라 달라지는 합의 모양

위치:
`calculus/riemann-integral-scaling/index.html`

대상 학생:
고등학교 미적분에서 구분구적법, 정적분의 정의, 치환적분의 기초 직관을 처음 배우는 일반계 고등학교 학생

수업 상황:
교사가 `구간을 잘게 자르면 넓이에 가까워진다`는 기본 설명을 한 뒤, 학생들이 자주 헷갈리는 다음 질문을 다룬다.

- 왜 `[0, 3]`을 `n`등분하면 `x_k = 3k/n`이 되는가?
- 왜 폭은 `1/n`이 아니라 `3/n`인가?
- `k/n`을 그냥 `x`라고 두면 무엇이 달라지는가?
- `x = 3t`처럼 변수를 바꾸면 왜 폭도 `Delta x = 3 Delta t`처럼 바뀌어야 하는가?
- 구분구적합은 어떻게 정적분 기호로 바뀌는가?

학습 목표:
- 구간 `[a, b]`를 `n`등분할 때 작은 구간의 폭이 `Delta x = (b - a) / n`임을 이해한다.
- 오른쪽 끝점, 왼쪽 끝점, 중점에 따라 표본점 `x_k`가 달라짐을 이해한다.
- 구분구적합 `sum f(x_k) Delta x`가 작은 직사각형 넓이들의 합임을 그래프와 수식으로 연결한다.
- `[0, 3]`에서 `x_k = 3k/n`, `Delta x = 3/n`이 자연스럽게 나온다는 것을 확인한다.
- `t_k = k/n`, `x_k = 3t_k`로 표현할 때 `Delta x = 3 Delta t`가 함께 따라와야 함을 이해한다.
- `x_k = k/n`, `Delta x = 1/n`은 `[0, 3]`의 넓이가 아니라 `[0, 1]`의 넓이를 구하는 표현임을 구분한다.
- `lim S_n = int_a^b f(x) dx`라는 정적분 정의로 넘어가는 과정을 단계적으로 설명할 수 있다.

핵심 개념:
- 구분구적법
- 리만 합
- 분할 개수 `n`
- 작은 구간의 폭 `Delta x`
- 표본점 `x_k`
- 오른쪽 끝점, 왼쪽 끝점, 중점 근사
- 구간 `[0, 3]`과 단위 구간 `[0, 1]`의 차이
- 매개변수 `t`를 이용한 표현
- `x = 3t`일 때 `Delta x = 3 Delta t`
- 정적분의 정의
- 치환적분으로 이어지는 직관

## 기본 함수와 예시

초기 기본값은 다음으로 둔다.

- 함수: `f(x) = x^2`
- 구간: `[0, 3]`
- 분할 개수: `n = 6`
- 표본점 방식: 오른쪽 끝점
- 표현 모드: `[0, 3] 직접 분할`

`f(x)=x^2`를 기본값으로 쓰는 이유:

- 그래프와 직사각형 넓이가 시각적으로 잘 드러난다.
- `[0, 3]`에서 정확한 넓이 `int_0^3 x^2 dx = 9`를 고등학생 수준에서 확인할 수 있다.
- `f(3k/n)`과 `(3k/n)^2`의 차이를 설명하기 쉽다.

추가 함수 선택지는 많지 않게 둔다.

- `f(x) = x^2`
- `f(x) = x`
- `f(x) = 1 + 0.5x^2`

직접 함수 입력 기능은 구현하지 않는다. 수업용 안정성을 우선한다.

## 표현 모드별 동작

### A. `[0,3] 직접 분할`

목표:
실제 구간 `[0,3]`을 직접 `n`등분하는 상황을 보여준다.

오른쪽 끝점 기준:

```text
Delta x = 3 / n
x_k = 3k / n
S_n = sum_{k=1}^{n} f(3k/n) * 3/n
```

왼쪽 끝점 기준:

```text
Delta x = 3 / n
x_k = 3(k-1) / n
S_n = sum_{k=1}^{n} f(3(k-1)/n) * 3/n
```

중점 기준:

```text
Delta x = 3 / n
x_k = 3(k - 1/2) / n
S_n = sum_{k=1}^{n} f(3(k - 1/2)/n) * 3/n
```

일반 구간 `[a,b]`에서는 다음으로 표시한다.

```text
Delta x = (b - a) / n
x_k = a + k Delta x
S_n = sum_{k=1}^{n} f(x_k) Delta x
```

그래프:
직사각형은 `[0,3]` 전체에 걸쳐 표시된다.

해석 문장:
`[0,3] 전체를 n등분했으므로 작은 폭은 3/n입니다. k번째 오른쪽 끝점은 x_k = 3k/n입니다.`

### B. `[0,1]의 t로 [0,3] 표현`

목표:
단위 구간 `[0,1]`에서 `t_k = k/n`을 잡되, 실제 그래프의 가로 위치는 `x = 3t`라는 점을 보여준다.

오른쪽 끝점 기준:

```text
Delta t = 1 / n
t_k = k / n
x_k = 3t_k = 3k / n
Delta x = 3 Delta t = 3 / n
S_n = sum_{k=1}^{n} f(3t_k) * 3 Delta t
S_n = sum_{k=1}^{n} f(3k/n) * 3/n
```

일반 구간 `[a,b]`에서는:

```text
t_k = k / n
x_k = a + (b - a)t_k
Delta x = (b - a) Delta t
S_n = sum_{k=1}^{n} f(a + (b - a)t_k) * (b - a) Delta t
```

그래프:
A와 같은 직사각형이 그려져야 한다. 즉, `[0,3]` 전체에 직사각형이 생긴다.

아래에는 A와 B가 같은 직사각형을 만든다는 표시를 둔다.

해석 문장:
`t는 0에서 1까지 움직이지만, 실제 x축 위치는 x=3t입니다. 그래서 폭도 Delta x = 3 Delta t가 되어야 같은 넓이를 구합니다.`

중요:
B 모드에서 직사각형 폭을 `1/n`으로 그리면 안 된다. 실제 그래프의 가로축은 `x`이므로 폭은 `3/n`이다.

### C. `[0,1]을 x로 분할`

목표:
`x_k = k/n`, `Delta x = 1/n`은 틀린 식이라기보다 `[0,1]`이라는 다른 구간의 넓이를 구하는 식임을 보여준다.

오른쪽 끝점 기준:

```text
Delta x = 1 / n
x_k = k / n
S_n = sum_{k=1}^{n} f(k/n) * 1/n
```

그래프:
직사각형은 `[0,1]` 구간에만 표시한다.

비교 표시:

- A, B: `[0,3]`의 넓이
- C: `[0,1]`의 넓이

해석 문장:
`x_k = k/n, Delta x = 1/n으로 두면 x축에서 실제로 [0,1]만 나눈 것입니다. [0,3]의 넓이를 구하려면 x_k 또는 Delta x에 구간 길이 3이 반영되어야 합니다.`

## 현재 직사각형 하나의 해석 패널

`k`번째 직사각형을 선택하면 다음 정보를 보여준다.

예시: `[0,3]`, `n=6`, 오른쪽 끝점, `k=4`, `f(x)=x^2`

```text
k = 4
Delta x = 3 / 6 = 0.5
x_4 = 3 * 4 / 6 = 2
높이 = f(2) = 4
작은 넓이 = f(2) * 0.5 = 2
```

B 모드에서는 같은 직사각형에 대해 다음도 함께 보여준다.

```text
t_4 = 4 / 6
x_4 = 3t_4 = 2
Delta t = 1 / 6
Delta x = 3 Delta t = 0.5
```

C 모드에서는 다음처럼 다르게 표시한다.

```text
x_4 = 4 / 6
Delta x = 1 / 6
이 직사각형은 [0,1] 안에 있습니다.
```

## 전체 합과 정적분 변환 패널

수식은 MathJax CDN 사용을 권장한다. 이유는 분수, 합, 극한, 정적분 기호를 고등학생이 읽기 좋게 보여줘야 하기 때문이다.

MathJax는 이 웹앱 하나에만 추가한다. CDN을 쓰므로 인터넷 연결이 없으면 수식 렌더링이 제한될 수 있음을 코드 주석에 짧게 남긴다.

패널은 `단계 보기` 버튼으로 1단계씩 펼쳐지게 한다.

### A 모드 단계

1단계:

```text
S_n = sum_{k=1}^{n} f(x_k) Delta x
```

2단계:

```text
Delta x = (b - a) / n
```

3단계:

```text
x_k = a + k Delta x
```

4단계:

```text
S_n = sum_{k=1}^{n} f(a + k Delta x) Delta x
```

5단계:

`[0,3]`일 때:

```text
S_n = sum_{k=1}^{n} f(3k/n) * 3/n
```

6단계:

`f(x)=x^2`일 때:

```text
S_n = sum_{k=1}^{n} (3k/n)^2 * 3/n
```

7단계:

```text
lim_{n -> infinity} S_n = int_0^3 f(x) dx
```

### B 모드 단계

1단계:

```text
t_k = k/n, Delta t = 1/n
```

2단계:

```text
x = 3t
```

3단계:

```text
x_k = 3t_k
```

4단계:

```text
Delta x = 3 Delta t
```

5단계:

```text
S_n = sum_{k=1}^{n} f(3t_k) * 3 Delta t
```

6단계:

```text
lim_{n -> infinity} S_n = int_0^1 f(3t) * 3 dt
```

7단계:

```text
int_0^1 f(3t) * 3 dt = int_0^3 f(x) dx
```

해석 문장:
`x=3t로 가로축을 3배 늘렸으므로, 작은 폭도 3배 늘어납니다. 이 3이 빠지면 넓이가 맞지 않습니다.`

### C 모드 단계

1단계:

```text
x_k = k/n, Delta x = 1/n
```

2단계:

```text
S_n = sum_{k=1}^{n} f(k/n) * 1/n
```

3단계:

```text
lim_{n -> infinity} S_n = int_0^1 f(x) dx
```

4단계:

```text
따라서 이 식은 [0,3]이 아니라 [0,1]의 넓이를 나타냅니다.
```

## 상태/결과 표시

항상 다음 값을 보여준다.

- 현재 모드
- 현재 함수
- 현재 구간
- `n`
- `Delta x`
- 선택된 표본점 방식
- 리만 합 `S_n`
- 기준 정적분 값
- 오차 `|S_n - integral|`

단, C 모드에서는 기준 정적분도 `[0,1]`에 맞춘 값으로 표시한다. 동시에 `현재 비교 대상은 [0,1]입니다`라는 문장을 둔다.

정확한 정적분 값은 선택 함수에 대해 직접 계산한다.

- `f(x)=x^2`: `b^3/3 - a^3/3`
- `f(x)=x`: `b^2/2 - a^2/2`
- `f(x)=1+0.5x^2`: `(b-a) + (b^3-a^3)/6`

## 사용자 조작에 따른 즉시 변화

`n` 변경:

- 직사각형 개수 변경
- 폭 `Delta x` 변경
- 리만 합 `S_n` 변경
- 오차 변경
- 현재 `k`가 `n`보다 커지면 자동으로 `k=n`으로 조정

구간 변경:

- 그래프 표시 범위 변경
- `Delta x = (b-a)/n` 변경
- 표본점 위치 변경
- 합과 정적분 값 변경

표본점 방식 변경:

- 직사각형 높이 변경
- 표본점 점 위치 변경
- 합과 오차 변경
- 수식의 `x_k` 표현 변경

표현 모드 변경:

- A와 B는 같은 직사각형이 유지되어야 한다.
- A와 B는 같은 `S_n` 값을 가져야 한다.
- C는 직사각형이 `[0,1]`로 이동하고, `S_n` 값도 달라져야 한다.
- 해석 문장과 수식 단계가 모드에 맞게 바뀌어야 한다.

현재 `k` 변경:

- 강조 직사각형 변경
- `k`번째 표본점과 작은 넓이 해석 변경

초기화:

- 함수 `x^2`

## 코드 작성 기준

- HTML, CSS, JavaScript를 하나의 `index.html` 안에 작성한다.
- 공통 스타일은 `../../assets/common.css`를 연결한다.
- 개별 앱 전용 스타일은 해당 HTML의 `<style>`에 둔다.
- 수학 계산 함수와 화면 렌더링 함수를 분리한다.
- 상태 객체를 하나 두고, 모든 조작은 상태 변경 후 `render()`를 호출하는 방식으로 구현한다.
- 자동 실행 기능은 넣지 않아도 된다. 이 앱은 반복 실험보다 개념 비교가 핵심이다.
- `n`, `k`, 구간 값이 꼬이지 않도록 입력값을 항상 보정한다.
- 로컬 절대경로를 사용하지 않는다.
- MathJax CDN 외의 외부 라이브러리는 사용하지 않는다.

## 완료 기준

- `[0,3] 직접 분할` 모드에서 `x_k = 3k/n`, `Delta x = 3/n`이 그래프와 수식에 일관되게 표시된다.
- `[0,1]의 t로 [0,3] 표현` 모드에서 A 모드와 같은 직사각형, 같은 합이 표시된다.
- B 모드에서 `Delta x = 3 Delta t`가 명확히 표시된다.
- `[0,1]을 x로 분할` 모드에서 직사각형이 `[0,1]`에만 생기고, `[0,3]`과 다른 넓이임이 드러난다.
- `n`을 키우면 직사각형이 곡선 아래 넓이에 가까워지는 모습과 오차 감소가 보인다.
- 왼쪽 끝점, 오른쪽 끝점, 중점 방식이 각각 다른 근사를 만들고, `n`이 커질수록 같은 정적분 값에 가까워지는 모습이 보인다.
- 현재 `k`번째 직사각형의 폭, 높이, 작은 넓이를 학생이 읽을 수 있다.
- 수식 변형 단계가 `sum f(x_k) Delta x`에서 정적분까지 자연스럽게 이어진다.
- 과목별 목록 페이지에서 새 웹앱으로 연결되는 링크가 추가된다.
- 실제 파일이 없는 링크를 만들지 않는다.
- GitHub Pages에서 상대경로로 바로 실행된다.

## 검증 방법

브라우저에서 다음을 확인한다.

1. `calculus/riemann-integral-scaling/index.html`을 직접 열었을 때 화면이 깨지지 않는다.
2. `calculus/index.html`에서 새 카드 링크를 눌렀을 때 웹앱이 열린다.
3. `n` 슬라이더를 움직이면 직사각형 개수, 합, 오차, 수식이 즉시 바뀐다.
4. 표본점 방식을 바꾸면 직사각형 높이와 `x_k` 식이 바뀐다.
5. A와 B 모드는 같은 직사각형과 같은 `S_n` 값을 보여준다.
6. C 모드는 `[0,1]` 구간만 직사각형으로 표시한다.
7. `[0,3]`, `f(x)=x^2`, `n`이 충분히 클 때 합이 `9`에 가까워진다.
8. B 모드의 정적분 변환에서 `int_0^1 f(3t) * 3 dt = int_0^3 f(x) dx`가 보인다.
9. 브라우저 콘솔에 JavaScript 오류가 없다.
10. 모바일 폭에서도 그래프, 조작 패널, 수식 패널이 겹치지 않는다.

## 수업 진행 예시

1. A 모드에서 `n=3`으로 시작한다.
2. `n`을 6, 12, 30으로 늘리며 직사각형 합이 넓이에 가까워지는 모습을 본다.
3. `k`를 움직이며 직사각형 하나의 넓이가 `f(x_k) Delta x`임을 확인한다.
4. 수식 단계를 펼쳐 `S_n = sum f(3k/n) * 3/n`까지 만든다.
5. B 모드로 바꿔 그림이 변하지 않는다는 것을 확인한다.
6. `t_k = k/n`이지만 실제 위치는 `x_k = 3t_k`이고, 폭은 `Delta x = 3 Delta t`임을 확인한다.
7. C 모드로 바꿔 `x_k = k/n`, `Delta x = 1/n`이 `[0,1]`의 넓이를 구한다는 것을 비교한다.
8. 마지막으로 `n -> infinity`에서 A와 B가 같은 정적분으로 연결됨을 정리한다.

## 구현자가 특히 주의할 점

- 이 웹앱의 핵심은 많은 기능이 아니라 A, B, C 모드의 차이를 선명하게 보여주는 것이다.
- A와 B는 같은 넓이를 다른 문자로 표현한 것이므로 그래프 결과가 같아야 한다.
- C는 틀린 식이라고 단정하지 말고, 다른 구간의 정적분이라고 설명한다.
- `x = 3t`를 보여줄 때 `Delta x = 3 Delta t`를 반드시 함께 보여준다.
- `dx = 3dt`라는 표현은 최종 단계에서만 연결하고, 먼저 구분구적법의 작은 폭 변화로 설명한다.
- 학생이 처음 보는 화면에서는 `n`, 모드, 직사각형 하나, 전체 합의 관계가 바로 보이게 한다.
