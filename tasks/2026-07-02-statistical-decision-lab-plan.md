# 통계적 의사결정 도우미 계획서

웹앱 이름:
통계적 의사결정 도우미

위치:
`probability-statistics/statistical-decision-lab/index.html`

대상 자료:
`lesson-design/probability-statistics/activities/statistical-decision-pilot/v3_스토리심화형`

우선 MVP:
`04 센서 경보 후 조치 결정` 학생용 PDF를 내장한 조건부확률·기저율 보조 도구

## 1. 계획의 핵심

이 웹앱은 학습지의 답을 대신 제시하는 계산기가 아니라, 학생이 학습지에서 이미 하고 있는 사고를 옆에서 받쳐 주는 자료다.

역할 분담은 다음처럼 둔다.

```text
학습지: 상황 이해, 예상, 계산 기록, 판단, 결론, 돌아보기
웹앱: PDF 보기, 위치 안내, 분모 변화 시각화, 자연빈도 그림, 간단 계산 확인
```

웹앱의 성공 기준은 학생이 빨리 답을 얻는 것이 아니다. 학생이 다음 구분을 자신의 말로 할 수 있으면 성공이다.

```text
센서 성능 질문: 위험할 때 경보가 울리는가?
의사결정 질문: 경보가 울렸을 때 진짜 위험인가?
```

## 2. 학습상 위험과 대응

웹앱을 항상 함께 쓰는 구조에서는 학생의 시선이 종이 학습지에서 화면으로 옮겨가며 사고가 끊길 수 있다. 또한 계산 결과가 화면에 너무 쉽게 나타나면 학생이 자신의 계산과 판단을 건너뛸 수 있다.

따라서 웹앱은 다음 원칙을 지킨다.

- 학습지 문항 전문을 웹앱에 옮기지 않는다.
- `정답`, `최선`, `추천`, `따라서` 같은 결론형 표현을 쓰지 않는다.
- 계산 결과는 기본 화면에 모두 펼쳐 두지 않고, `내 계산 확인`처럼 학생 행동 뒤에 보여준다.
- Q1~Q5 답을 한 번에 채워 주는 표를 만들지 않는다.
- AI 요약 비판 문항(Q8)의 정답이나 반박 문장을 제공하지 않는다.
- 계산 과정 설명은 학습지에 제시된 수준을 넘지 않는다.
- 화면 전환이 필요하면 `학습지 Q2 참고`, `학습지 Q4로 돌아가기`처럼 짧은 위치 안내만 제공한다.

## 3. MVP 주제 선정

첫 MVP는 `04 센서 경보 후 조치 결정`으로 한다.

선정 이유:

- 조건부확률의 대표 오개념인 `P(경보 | 위험)`과 `P(위험 | 경보)`의 방향 혼동을 다룬다.
- 자연빈도와 2×2 빈도표로 분모가 바뀌는 장면을 보여주기 좋다.
- 후속 `01 상품추천 규칙`과 같은 조건부확률 모델을 공유한다.

`01 상품추천 규칙`은 MVP 화면에 preset 버튼만 둔다. 실제 계산과 시각화는 후속 작업으로 보류한다.

## 4. 센서 경보 학습지 기준값

MVP 기본값은 실제 학생용 학습지와 일치해야 한다.

```text
전체 건수 N = 10,000
기저율 P(위험) = 1%
위험 건수 = 100
정상 건수 = 9,900
민감도 P(경보 | 위험) = 95%
오경보율 P(경보 | 정상) = 5%
```

센서 1개일 때 자연빈도:

```text
위험이고 경보 = 95
위험인데 경보 없음 = 5
정상인데 경보 = 495
정상이고 경보 없음 = 9,405
경보 전체 = 590
P(위험 | 경보) = 95 / 590 ≈ 16.1%
```

두 센서가 모두 경보일 때는 Q5에서 다룬다. MVP에서 이 값을 다룰 경우에도 정답 제시가 아니라 `계산 확인`으로만 둔다.

```text
위험이고 두 센서 모두 경보 = 100 × 0.95² = 90.25
정상인데 두 센서 모두 경보 = 9,900 × 0.05² = 24.75
P(위험 | 두 센서 모두 경보) = 90.25 / (90.25 + 24.75) ≈ 78.5%
```

## 5. 사용 흐름

기본 흐름:

```text
1. 학습지 [A] 상황과 [B] 임무를 읽는다.
2. 학습지 [시작 전] 예상을 먼저 적는다.
3. 웹앱에서 센서 경보 preset과 PDF를 확인한다.
4. 학습지 Q1을 풀며 자연빈도 그림과 빈도표를 필요할 때 참고한다.
5. Q2에서 두 조건부확률의 분모 차이를 확인한다.
6. Q4 이후 판단 문항은 학습지로 돌아가 직접 쓴다.
```

웹앱에 넣을 수 있는 짧은 지시문:

- `학습지 [시작 전] 예상에 먼저 적고 확인하세요.`
- `학습지 Q1 빈도표를 채운 뒤 내 계산을 확인하세요.`
- `학습지 Q2에서 두 확률의 분모를 비교하세요.`
- `학습지 Q4로 돌아가 판단 근거를 직접 쓰세요.`

## 6. 화면 배치 계획

### iPad A16 가로 화면

최우선 검증 viewport:
`1180×820` CSS px

가로 화면 기본 배치:

```text
왼쪽 45%: 학생용 PDF
오른쪽 55%: 센서 경보 도구
```

오른쪽 도구의 밀도는 낮게 유지한다.

```text
상단: preset 버튼과 학습지 위치 안내
중간: 자연빈도 그림 또는 비율 막대
중간: 2×2 빈도표와 분모 강조
하단: 내 계산 확인, 학습지로 돌아가는 문구
```

### iPad A16 세로 화면

보조 검증 viewport:
`820×1180` CSS px

세로 화면에서는 좌우 분할 대신 탭 전환을 쓴다.

```text
[학습지 보기] [도구 보기]
```

기본 탭은 `학습지 보기`로 둔다. 학생이 도구를 열어야 할 때는 도구 상단 안내가 학습지 Q번호를 짧게 가리킨다.

## 7. 시각화 전략

센서 경보 MVP는 수식보다 자연빈도와 분모 이동을 먼저 보여준다.

```text
전체 10,000건
-> 실제 위험 100건 / 정상 9,900건으로 나눔
-> 위험 중 경보 95건, 정상 중 경보 495건을 표시
-> 경보가 울린 590건만 다시 모음
-> 그 안에서 위험 95건의 비율을 확인
```

핵심 시각화:

- 전체 사례 막대 또는 격자
- 실제 위험/정상 구분
- 경보/경보 없음 구분
- 2×2 빈도표
- `P(경보 | 위험)`과 `P(위험 | 경보)` 비교

가장 중요한 장치는 `분모 강조`다.

- `P(경보 | 위험)`을 볼 때는 위험 100건을 분모로 강조한다.
- `P(위험 | 경보)`를 볼 때는 경보 전체 590건을 분모로 강조한다.
- 색만으로 구분하지 않고 라벨, 굵은 테두리, 짧은 문구를 함께 쓴다.

## 8. 계산 기능 범위

계산 기능은 `내 계산 확인`으로 제한한다.

포함 가능:

- 빈도표 네 칸 계산 확인
- 경보 전체 건수 확인
- `P(경보 | 위험)`과 `P(위험 | 경보)` 확인
- 두 센서 모두 경보인 경우의 Q5 계산 확인

피할 것:

- Q1~Q5의 모든 빈칸을 한 번에 채우는 기능
- 대응안 가/나/다 중 하나를 자동 선택하는 기능
- `AI 요약에서 틀린 부분`에 대한 완성 문장 제공
- 계산 원리를 학습지보다 더 자세히 풀어 주는 설명

## 10. 사전 검토 판정

학습가치:
통과. 조건부확률의 방향 혼동을 자연빈도와 분모 강조로 보조하므로 학습지의 사고 흐름을 해치지 않고 보완할 수 있다.

수식 표현:
통과. MVP는 복잡한 수식 렌더링보다 빈도표, 분모, 문장 방향 구분이 핵심이므로 MathJax 없이 구현 가능하다.

화면 배치:
부분 수정 후 통과 가능. PDF와 도구를 함께 두는 구조는 타당하지만, 실제 계산 도구가 들어간 뒤 iPad 세로 탭 전환, PDF 높이, 도구 영역 스크롤을 다시 검증해야 한다.

## 11. 완료 기준

- `probability-statistics/statistical-decision-lab/index.html`이 GitHub Pages 정적 파일로 열린다.
- `probability-statistics/index.html`에 실제 페이지로 연결되는 카드가 있다.
- 학생용 PDF만 웹앱 폴더에 있으며, 화면에서 볼 수 있고 다운로드할 수 있다.
- 교사용 PDF, 교사용 Markdown, 예시답안 파일은 웹앱 폴더에 없다.
- 센서 경보 preset이 활성화되어 있고 상품추천 preset 버튼은 준비 중 상태로 보인다.
- 센서 경보 기본값이 학습지와 일치한다.
- 웹앱은 최종 판단이나 권고안을 제시하지 않는다.
- iPad A16 가로 `1180×820`에서 PDF와 도구가 함께 보인다.
- iPad A16 세로 `820×1180`에서 학습지/도구 전환이 가능하다.
