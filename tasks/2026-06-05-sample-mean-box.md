# 상자에서 뽑는 표본평균 웹앱 작업지시서

웹앱 이름:
상자에서 뽑는 표본평균

위치:
`probability-statistics/sample-mean-box/index.html`

관련 웹앱:
`probability-statistics/sample-mean/index.html`

이 작업지시서는 기존 `tasks/2026-06-05-sample-mean.md`를 대체하지 않는다. 기존 작업지시서는 표본평균의 분포와 표준오차를 확인하는 실험실이고, 이 작업지시서는 표본평균 하나가 만들어지고 기록되는 과정을 시각적으로 보여주는 도입용 웹앱이다.

검토 후 보강 방향:
이 앱은 `표본평균의 분포` 자체보다 `표본평균 하나가 만들어지는 과정`을 우선한다. 따라서 구현자는 화려한 분포 그래프보다 "공 n개가 표본 바구니에 모임 -> 평균 계산 -> xbar 하나가 기록판에 추가됨"이라는 원인과 결과가 끊기지 않게 보이는 것을 최우선으로 한다.

대상 학생:
고등학교 확률과 통계 과목에서 모집단, 표본, 표본평균, 표본평균들의 평균을 처음 배우는 일반계 고등학교 학생

수업 상황:
교사는 "상자 안의 번호 공에서 몇 개를 뽑아 평균을 내면, 그 평균은 무엇을 대표할까?"라는 질문으로 수업을 시작한다. 학생은 상자에서 공이 실제로 뽑히고, 뽑힌 공들의 평균이 계산되어 기록판에 하나의 값으로 남는 과정을 보면서 표본평균의 의미를 이해한다.

학습 목표:
- 모집단을 "번호가 적힌 공들이 들어 있는 상자"로 이해한다.
- 한 번의 표본 추출은 상자에서 `n`개의 공을 뽑는 과정임을 이해한다.
- 한 표본에서 나온 `n`개의 값을 평균 내면 표본평균 `xbar` 하나가 만들어짐을 확인한다.
- 표본평균 하나가 기록판에 점 또는 카드 하나로 기록되는 과정을 본다.
- 표본을 여러 번 뽑으면 표본평균도 여러 개 쌓이고, 그 표본평균들의 평균을 계산할 수 있음을 이해한다.
- `표본 크기 n`과 `반복 추출 횟수`를 구분한다.

핵심 개념:
- 모집단
- 표본
- 표본 크기 `n`
- 표본평균 `xbar`
- 표본평균들의 평균
- 반복 추출
- 복원 개념

## 설계 방향

이 웹앱은 표본평균의 분포를 정교하게 분석하는 앱이 아니라, 표본평균이 만들어지는 과정을 애니메이션으로 보여주는 개념 도입용 앱이다.

핵심 장면:

```text
상자 속 번호 공들 -> n개 뽑기 -> 표본 바구니에 모으기 -> 평균 계산 -> xbar 하나를 기록판에 추가
```

학생이 반드시 보아야 하는 연결:

1. 상자 안에는 모집단 값들이 들어 있다.
2. 표본 하나는 공 `n`개로 이루어진다.
3. 공 `n`개의 평균을 내면 `xbar` 하나가 나온다.
4. 기록판에는 원자료 공들이 아니라 `xbar` 값 하나가 기록된다.
5. 이 과정을 여러 번 반복하면 기록판에 `xbar`가 여러 개 쌓인다.

## 기존 표본평균 분포 앱과의 차이

기존 작업지시서 `tasks/2026-06-05-sample-mean.md`의 앱:

- 분포 선택
- 히스토그램 3개
- 표준오차 `sigma / sqrt(n)` 비교
- 중심극한정리 관찰
- 많은 반복 추출을 통한 분포 형성

이번 앱:

- 상자와 번호 공을 중심으로 한 구체적 비유
- 추출 애니메이션
- 표본 바구니와 평균 계산 과정
- `xbar` 하나가 기록되는 장면 강조
- 표본평균들의 평균을 직관적으로 이해

이번 앱에서는 중심극한정리와 표준오차 설명을 깊게 다루지 않는다. 필요하면 마지막 해석 문구에서 "더 많이 반복하면 표본평균들이 분포를 만든다" 정도로만 연결한다.

특히 이번 앱에서 `표본평균들의 평균`은 기록판에 쌓인 `xbar` 값들의 산술평균이다. 표본 개수가 적을 때는 이 값이 모집단 평균과 꽤 다를 수 있으며, 이 흔들림도 자연스러운 관찰 대상으로 둔다.

## 사용자 조작

### 모집단 상자 선택

`select` 또는 버튼 그룹으로 제공한다.

- 균형 잡힌 상자
  - 예: `60, 65, 70, 75, 80, 85, 90, 95, 100` 값이 고르게 섞인 공
- 낮은 값이 많은 상자
  - 예: 60~75 값이 많고 90~100 값이 적은 공
- 높은 값이 많은 상자
  - 예: 85~100 값이 많고 60~75 값이 적은 공
- 양쪽으로 나뉜 상자
  - 예: 60~70 근처와 90~100 근처 값이 많은 공

각 상자는 `30`개 또는 `40`개의 공으로 구성한다. 너무 많은 공을 실제 DOM 요소로 모두 움직이면 화면이 복잡해지므로, 상자 안에는 대표 공들을 보이게 하고 전체 개수는 숫자로 표시해도 된다.

구현 시 각 공은 `id`와 `value`를 가진 객체로 둔다.

```text
ball = { id: "balanced-01", value: 75 }
```

같은 숫자 값을 가진 공이 여러 개 있을 수 있다. 한 표본을 뽑는 동안에는 같은 `id`의 공이 다시 뽑히지 않게 하고, 표본 하나가 끝나면 모든 공을 다시 뽑을 수 있게 한다.

### 모집단 평균 표시

기본적으로 모집단 평균 `μ`를 보여준다. 단, 수업 상황에 따라 학생에게 먼저 추측하게 하고 싶을 수 있으므로 작은 체크박스나 토글을 둘 수 있다.

- `μ 보이기`: 기본 켜짐
- 꺼졌을 때는 모집단 평균 숫자와 기준선을 숨기고, "모집단 평균은 숨김"이라고 표시한다.

이 토글은 선택 기능이다. 구현 시간이 부족하면 항상 보이게 해도 된다.

### 표본 크기 `n`

- 슬라이더와 빠른 선택 버튼을 함께 둔다.
- 범위: `2`부터 `10`
- 기본값: `5`
- 빠른 선택:
  - `n = 2`
  - `n = 3`
  - `n = 5`
  - `n = 8`
  - `n = 10`

이 앱은 추출 애니메이션이 핵심이므로 `n`을 너무 크게 잡지 않는다.

설명 문구:

```text
n은 한 번에 뽑아 표본을 만들 공의 개수입니다.
```

### 실행 버튼

- `표본 1개 뽑기`
- `천천히 1개씩 뽑기`
- `표본 5개 연속 뽑기`
- `자동 반복`
- `일시정지`
- `초기화`

버튼 의미:

- `표본 1개 뽑기`: 공 `n`개를 애니메이션으로 뽑고, `xbar` 하나를 기록한다.
- `천천히 1개씩 뽑기`: 공 하나씩 수동으로 뽑아 표본 바구니가 채워지는 과정을 더 천천히 본다.
- `표본 5개 연속 뽑기`: 표본 5개를 차례로 만들고, `xbar` 5개를 기록한다.
- `자동 반복`: 표본 만들기와 기록을 반복한다.
- `일시정지`: 자동 반복을 멈춘다.
- `초기화`: 기록된 표본평균과 현재 표본을 모두 지운다.

자동 반복 중 중복 타이머가 생기지 않게 한다.

`천천히 1개씩 뽑기`는 상태가 있는 버튼으로 구현한다.

- 현재 표본이 없으면 새 표본을 시작하고 첫 공 1개를 뽑는다.
- 현재 표본이 진행 중이면 다음 공 1개를 뽑는다.
- `n`개가 채워지는 순간 평균 계산판을 보여주고 `xbar` 하나를 기록한다.
- 표본이 완성된 뒤 다시 누르면 새 표본을 시작한다.

수동 추출 중에는 모집단 상자 선택, 표본 크기 변경, 자동 반복 시작을 막거나, 변경 시 현재 표본이 초기화된다는 확인 없이 즉시 초기화하지 않는다. 수업 중 실수 클릭을 줄이기 위해 진행 중에는 `초기화`와 현재 추출 버튼만 활성화하는 편이 좋다.

### 애니메이션 속도

- 느림
- 보통
- 빠름

속도 조절은 버튼 또는 작은 선택 메뉴로 제공한다.

## 화면 구성

기본 구조:

```text
[상단 설명]
[조작 패널]
[상자 영역] -> [이번 표본 바구니] -> [평균 계산판] -> [표본평균 기록판]
[해석 문구]
```

### 상단 설명

- 제목: `상자에서 뽑는 표본평균`
- 설명:
  - "상자에서 번호 공을 뽑아 표본을 만들고, 그 표본의 평균이 하나의 값으로 기록되는 과정을 봅니다."
  - "한 번의 표본 추출에서 xbar는 하나만 생깁니다."

### 조작 패널

- 모집단 상자 선택
- 표본 크기 `n`
- 애니메이션 속도
- 실행 버튼들
- 현재 상태 표시
  - `이번 표본: 0 / n개`
  - `기록된 xbar: 0개`

### 상자 영역

- 번호가 적힌 공들이 들어 있는 상자를 시각화한다.
- 공에는 숫자가 크게 표시되어야 한다.
- 공은 색만으로 구분하지 말고 숫자를 반드시 표시한다.
- 상자 아래에 모집단 평균을 표시한다.
- 한 표본을 뽑는 중 이미 뽑힌 공은 상자 안에서 흐리게 처리하거나 "이번 표본에서 뽑힘" 상태로 표시한다.
- 표본이 기록된 뒤에는 공이 다시 상자로 돌아온 것처럼 원래 상태로 복원한다.

예:

```text
모집단 평균 μ = 82.4
상자 안 공 개수 = 40개
```

### 이번 표본 바구니

- `n`개의 빈 칸을 만든다.
- 공이 뽑힐 때마다 빈 칸에 하나씩 들어간다.
- 각 칸에는 공의 숫자가 보인다.
- 바구니 아래에 현재 합계와 현재 평균을 표시한다.

예:

```text
현재 합계: 410
현재 평균: 82.0
```

표본이 완성되기 전에는 "현재 평균"이라고 표시하고, `n`개가 모두 뽑히면 "이번 표본평균 xbar"라고 바꾼다.

### 평균 계산판

표본이 완성되면 계산식을 크게 보여준다.

```text
xbar = (80 + 75 + 90 + 85 + 70) / 5 = 80.0
```

숫자가 너무 많을 때는 합계 형태로 줄인다.

```text
xbar = 400 / 5 = 80.0
```

계산판에서 나온 `xbar` 값이 기록판으로 이동하는 애니메이션을 넣는다. 실제 이동 애니메이션이 어렵다면 `xbar` 카드가 강조된 뒤 기록판에 추가되는 방식으로 표현해도 된다.

### 표본평균 기록판

기록판에는 각 표본에서 나온 `xbar`가 하나씩 쌓인다.

표현 방식:

- 점 그래프
  - 가로축: 표본평균 값
  - 점 하나: 표본 하나에서 나온 `xbar` 하나
- 또는 기록 카드 목록
  - `1번째 표본: xbar = 80.0`
  - `2번째 표본: xbar = 84.0`

가능하면 점 그래프와 최근 5개 기록 목록을 함께 제공한다.

점 그래프 규칙:

- 가로축 범위는 현재 모집단 값의 최솟값부터 최댓값까지로 고정한다.
- 새 점이 추가될 때마다 축 범위가 계속 바뀌어 기존 점이 흔들리는 일을 피한다.
- 같은 값 또는 비슷한 값의 `xbar`가 여러 개 생기면 점을 세로로 살짝 쌓거나 작은 흔들림을 주어 겹침을 줄인다.
- 최신 `xbar` 점은 잠깐 강조한다.

기록판에는 다음을 표시한다.

- 기록된 표본평균 개수
- 표본평균들의 평균
- 모집단 평균 `μ`
- 모집단 평균 기준선
- 표본평균들의 평균 기준선

## 애니메이션 흐름

구현은 명확한 상태값을 기준으로 한다.

권장 상태:

- `idle`: 진행 중인 표본 없음
- `manual-drawing`: 수동으로 공을 하나씩 뽑는 중
- `animating-sample`: 표본 1개 또는 연속 표본 애니메이션 중
- `recording`: 평균 계산 후 `xbar`를 기록판에 추가하는 중
- `auto-running`: 자동 반복 중

상태가 바뀔 때 버튼 활성/비활성 상태도 함께 갱신한다.

### 표본 1개 뽑기

1. 현재 표본 바구니를 비운다.
2. 상자에서 공 하나를 선택한다.
3. 선택된 공이 강조된다.
4. 공이 표본 바구니의 다음 빈 칸으로 이동한다.
5. 합계와 현재 평균이 갱신된다.
6. `n`개가 채워질 때까지 2~5를 반복한다.
7. 평균 계산판에 `xbar` 계산식을 표시한다.
8. `xbar` 카드 또는 점이 기록판으로 이동한다.
9. 기록판의 표본평균들의 평균이 갱신된다.
10. 표본 하나가 끝나면 공들은 상자로 돌아간 것으로 처리한다.

### 표본 5개 연속 뽑기

- 표본 5개를 한꺼번에 계산해서 한 번에 기록하지 않는다.
- 학생이 "표본 하나 -> xbar 하나"의 반복을 볼 수 있도록 5번을 순서대로 진행한다.
- 속도는 `표본 1개 뽑기`보다 조금 빠르게 해도 되지만, 각 표본의 `xbar`가 하나씩 기록되는 장면은 보여야 한다.

### 자동 반복

- 자동 반복은 표본 하나가 완성되고 기록된 뒤 다음 표본을 시작한다.
- 이전 표본이 기록판에 들어가기 전에 다음 표본을 시작하지 않는다.
- 자동 반복 중 `일시정지`를 누르면 현재 진행 중인 표본을 마저 기록한 뒤 멈추거나, 즉시 멈추되 화면에 "일시정지됨" 상태를 분명히 표시한다.
- 어떤 방식을 택하든 중간 상태가 깨지지 않아야 한다.

### 복원 처리

개념 혼동을 줄이기 위해 다음 규칙을 사용한다.

- 한 표본을 만드는 동안에는 같은 공이 중복으로 뽑히지 않게 한다.
- 표본 하나가 끝나면 뽑힌 공들은 모두 상자에 다시 들어간다.
- 따라서 다음 표본은 같은 모집단에서 다시 뽑는다.

화면 문구:

```text
표본 하나가 완성되면 공을 다시 상자에 넣고 다음 표본을 뽑습니다.
```

주의:
이 앱은 직관적 도입을 위해 "한 표본 안에서는 중복 없이 뽑고, 표본 사이에서는 다시 넣기"로 표현한다. 표준오차의 이론값을 다루는 앱이 아니므로 유한모집단 보정 같은 내용은 여기서 설명하지 않는다.

## 해석 문구

외부 AI 호출 없이 현재 상태에 따라 문구를 바꾼다.

처음 상태:

```text
상자 안 공들이 모집단입니다. 표본 1개를 뽑으면 공 n개가 바구니에 들어가고, 그 평균 xbar 하나가 기록됩니다.
```

표본을 뽑는 중:

```text
지금은 표본 하나를 만드는 중입니다. 공 n개가 모두 모여야 이번 표본평균 xbar가 완성됩니다.
```

표본 하나 완성:

```text
이번 표본의 n개 값을 평균 내어 xbar 하나를 만들었습니다. 이 값 하나가 오른쪽 기록판에 추가됩니다.
```

여러 표본 기록 후:

```text
기록판에는 원래 공들이 아니라 각 표본에서 나온 xbar 값들이 쌓입니다. 이 값들의 평균을 보면 표본평균들의 평균을 확인할 수 있습니다.
```

표본평균들의 평균이 모집단 평균에 가까울 때:

```text
표본평균들의 평균이 모집단 평균 μ에 가까워지고 있습니다. 표본을 반복해서 뽑으면 평균들이 μ 주변에 모이는 경향을 볼 수 있습니다.
```

표본평균들의 평균이 아직 모집단 평균과 차이가 클 때:

```text
아직 기록된 xbar가 많지 않아 표본평균들의 평균이 μ와 차이 날 수 있습니다. 표본을 더 반복해 보며 어떻게 변하는지 관찰해 봅니다.
```

## 수학 모델 또는 계산식

모집단:

```text
population = [공1의 값, 공2의 값, ..., 공N의 값]
```

모집단 평균:

```text
mu = sum(population) / N
```

한 표본:

```text
sample = [x1, x2, ..., xn]
```

표본평균:

```text
xbar = (x1 + x2 + ... + xn) / n
```

기록된 표본평균:

```text
sampleMeans = [xbar1, xbar2, ..., xbar_m]
```

표본평균들의 평균:

```text
meanOfSampleMeans = sum(sampleMeans) / m
```

여기서 `n`은 한 표본 안의 공 개수이고, `m`은 지금까지 만든 표본의 개수이다.

중요 구분:

```text
n = 한 표본을 만들 때 뽑는 공 개수
m = 기록판에 쌓인 xbar 개수
```

`표본평균들의 평균`은 `m`개의 `xbar`를 평균낸 값이다. 원자료 공 전체를 다시 평균낸 값이 아니다.

## 검증 방법

기본 동작:

- 페이지를 열었을 때 상자, 빈 표본 바구니, 빈 기록판이 보이는지 확인한다.
- 기본값이 `n = 5`인지 확인한다.
- `천천히 1개씩 뽑기`에서 공이 하나씩 바구니에 들어가는지 확인한다.
- `천천히 1개씩 뽑기`를 `n - 1`번 눌렀을 때는 아직 기록판에 새 `xbar`가 추가되지 않는지 확인한다.
- `천천히 1개씩 뽑기`를 `n`번째 눌렀을 때 `xbar`가 정확히 하나 추가되는지 확인한다.
- 공이 들어갈 때마다 합계와 현재 평균이 바뀌는지 확인한다.
- `n`개가 채워졌을 때 `xbar` 계산식이 표시되는지 확인한다.
- `xbar`가 기록판에 정확히 하나 추가되는지 확인한다.
- `표본 5개 연속 뽑기` 후 기록된 `xbar` 개수가 정확히 5 증가하는지 확인한다.

수치 검증:

- 바구니에 들어간 공 값의 합계가 화면의 합계와 일치하는지 확인한다.
- `xbar = 합계 / n`이 화면 표시와 일치하는지 확인한다.
- 표본을 5번 뽑으면 기록된 `xbar` 개수가 5인지 확인한다.
- 표본평균들의 평균이 기록된 `xbar` 값들의 산술평균과 일치하는지 확인한다.
- `m`은 기록된 `xbar` 개수이고, `n` 변경과 독립적으로 증가하는 값인지 확인한다.

상태 검증:

- 애니메이션 중 실행 버튼을 여러 번 눌러도 표본이 겹쳐서 생성되지 않는지 확인한다.
- 자동 반복 중 `일시정지`를 누르면 멈추는지 확인한다.
- 자동 반복 중 `초기화`를 누르면 타이머와 진행 중 표본이 모두 정리되는지 확인한다.
- 모집단 상자를 바꾸면 현재 표본과 기록판이 초기화되는지 확인한다.
- 표본 크기 `n`을 바꾸면 현재 표본과 기록판이 초기화되는지 확인한다.
- 한 표본을 뽑는 중 같은 `id`의 공이 두 번 뽑히지 않는지 확인한다.
- 표본 하나가 끝난 뒤에는 이전 표본에서 뽑힌 공도 다시 뽑힐 수 있는 상태가 되는지 확인한다.
