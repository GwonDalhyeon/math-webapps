# 통계적 의사결정 도우미 구현 지침서

이 문서는 `probability-statistics/statistical-decision-lab/`의 웹앱 전체를 구현하거나 수정할 때 따른다.

이 모음은 `lesson-design/.../v3_스토리심화형`의 학습지 `01`~`11`을 각각 보조하는 도구들의 묶음이다. 학습지는 주제는 달라도 다음 공통 골격을 공유한다.

```text
[시작 전] 예상 -> 핵심층 Q1~Q3 -> 판단·설계층 Q4~Q5 -> 심화·확장층 Q6~Q8(선택) -> [E] 결론 -> [F] 돌아보기
Q8은 대체로 "AI 요약에서 틀린 부분 찾기" 비판 문항이다.
```

그래서 지침은 두 층위로 나눈다.

- **A. 공통 지침** (§1~§10): 모든 앱이 그대로 따른다. 주제와 무관하다.
- **B. 모델별 부록** (§11): 학습지의 수학 모델 유형에 따라 §11에서 해당 항목만 골라 적용한다.
- **C. 완료·검증** (§12~§13): 공통 항목과 모델별 추가 항목으로 나눈다.

> 중요: 예전 지침은 `04 센서 경보` 하나만을 기준으로 작성돼 있었다. 조건부확률 전용이던 preset·계산 모델·분모 강조·두 센서 항목은 이제 §11-1(조건부확률 부록)로 옮겼다. 다른 유형의 앱에 조건부확률 전용 규칙(분모 강조 등)을 기계적으로 적용하지 않는다. 각 앱은 §11에서 자기 유형의 부록만 따른다.

### 11-1. 조건부확률·기저율 (04 센서경보, 01 상품추천)

핵심 오개념: `P(신호 | 사건)`과 `P(사건 | 신호)`의 방향 혼동. 분모가 바뀐다는 점을 자연빈도로 보여준다.

입력값:

- 전체 사례 수 `N`, 실제 사건 비율 `baseRate`, 사건일 때 신호 비율 `sensitivity`, 사건이 아닐 때 신호 비율 `falsePositiveRate`

핵심 계산:

```text
eventCount    = N × baseRate
nonEventCount = N - eventCount
truePositive  = eventCount × sensitivity
falseNegative = eventCount - truePositive
falsePositive = nonEventCount × falsePositiveRate
trueNegative  = nonEventCount - falsePositive
signalTotal   = truePositive + falsePositive
posterior     = truePositive / signalTotal
```

핵심 그림·강조:

- 전체 사례 막대/격자 -> 실제 사건/비사건 구분 -> 신호/비신호 구분 -> 2×2 빈도표
- 분모 강조: `P(신호 | 사건)`을 볼 때는 사건 건수를 분모로, `P(사건 | 신호)`를 볼 때는 신호 전체를 분모로 강조한다. 색·라벨·테두리·문구를 함께 쓴다.

입력형 계산 확인:

- 학생이 정한 분자·분모를 입력해 비율만 표시한다. 빈도표 네 칸·신호 전체·조건부확률 값을 버튼 한 번으로 공개하지 않는다.

두 신호(예: 두 센서 모두 경보) 보조 — 04 전용:

```text
사건이고 두 신호 모두 = eventCount × sensitivity²
비사건인데 두 신호 모두 = nonEventCount × falsePositiveRate²
P(사건 | 두 신호 모두) = 앞 항 / (두 항의 합)
```

주의: 구조(`100 × 0.95²`, `9,900 × 0.05²`)만 보여주고 각 빈칸을 미리 채우지 않는다. `추가 측정이 정답`이라 말하지 않고, 독립 가정 비판(Q7)의 답을 제공하지 않으며, 두 신호가 독립이라는 가정이 들어갔다는 짧은 라벨만 둔다.

04 센서경보 preset(학습지 일치값):

```text
N = 10,000, baseRate = 1%, sensitivity = 95%, falsePositiveRate = 5%
위험 100 / 정상 9,900
위험이고 경보 95, 위험인데 경보 없음 5
정상인데 경보 495, 정상이고 경보 없음 9,405
경보 전체 590,  P(위험 | 경보) = 95 / 590 ≈ 16.1%
labels: { event:"위험", nonEvent:"정상", signal:"경보", noSignal:"경보 없음" }
```

01 상품추천: 같은 조건부확률 모델을 공유한다. preset의 `N`·비율·라벨은 01 학습지 [C]에 맞춘다.

### 11-2. 표본흔들림·추정 (05 A/B실험, 08 학교시설, 11 설문표본)

핵심 오개념: 표본비율 `p̂`의 차이를 **표본이 바뀔 때의 흔들림 폭**과 비교하지 않고 바로 결론내기. 표본이 커지면 흔들림이 √n에 반비례로 줄어든다.

입력값:

- 표본 크기 `n`, 성공 수(또는 `p̂`), 비교 대상이 있으면 두 표본의 값

핵심 계산:

```text
phat = 성공 수 / n
흔들림(대략)  ≈ 2 × sqrt(phat × (1 - phat) / n)     // 학습지의 어림 폭
차이          = phat_B - phat_A
n을 k배로 키우면 흔들림은 약 1/sqrt(k) 배
```

핵심 그림·강조:

- 수직선/막대 위에 `p̂`와 그 흔들림 구간(`p̂ ± 흔들림`)을 그린다.
- 두 표본이 있으면 두 구간을 겹쳐 그려 **차이가 흔들림 안인지 밖인지**를 눈으로 비교하게 한다.
- `n` 슬라이더로 흔들림 폭이 좁아지는 과정을 보여준다. 강조 대상은 "흔들림 폭"과 "차이".

입력형 계산 확인:

- 학생이 `n`, 성공 수를 입력하면 `p̂`와 흔들림 폭만 표시한다.
- "차이가 흔들림보다 큰가/작은가"의 **판단 문장은 학생이 쓴다.** 앱은 두 수치만 나란히 보여준다.
- `n`을 몇 배로 키워야 흔들림이 절반이 되는지 같은 값은 학생이 입력한 목표에 대해서만 계산한다.

### 11-3. 기대손실·임계값 (03 행사준비, 10 우천행사)

핵심 오개념: "평균만큼 준비하면 충분하다"는 착각. 손실은 부족·초과의 **비대칭 비용**과 확률분포로 결정되며, 비용이 바뀌면 최적 준비량(임계값)도 바뀐다.

입력값:

- 분포 파라미터(예: `n`, `p`), 준비량/임계값 후보 `c`, 부족 1단위 손실, 초과 1단위 손실

핵심 계산:

```text
부족 확률   = P(X > c)              // 분포에서
평균 부족   = E[max(X - c, 0)]
평균 잔여   = E[max(c - X, 0)]
예상 손실 L(c) = 부족손실 × 평균부족 + 초과손실 × 평균잔여
```

핵심 그림·강조:

- 준비량 후보별 예상 손실을 막대/꺾은선으로 비교하고, **최소가 되는 지점(임계값)**을 강조한다.
- 손실 비용을 바꾸면 최적 지점이 이동하는 것을 보여준다. 강조 대상은 "임계값의 이동".

입력형 계산 확인:

- 학생이 평균 부족·평균 잔여·손실 비용을 입력하면 `L(c)`만 계산한다.
- 대응안 가/나/다 중 하나를 앱이 고르지 않는다. 세 값을 나란히 보여주고 선택은 학생이 한다.

### 11-4. 이항분포·반복시행 (02 우주통신, 06 불량품검사)

핵심 개념: `X ~ B(n, p)`. 반복시행에서 성공 수의 분포, 특정 구간의 확률, 시행 수를 늘릴 때의 변화.

입력값:

- 시행 수 `n`, 1회 성공 확률 `p`, 관심 사건의 경계(예: `X ≥ k`)

핵심 계산:

```text
P(X = k) = C(n,k) p^k (1-p)^(n-k)
누적확률 P(X ≥ k), P(X ≤ k)
E(X) = np,  표준편차 = sqrt(n p (1-p))
```

핵심 그림·강조:

- `B(n,p)` 막대그래프. 관심 구간(`X ≥ k` 등)을 색·테두리로 강조한다.
- `n` 또는 `p` 슬라이더로 분포 모양과 강조 구간의 확률이 변하는 과정을 보여준다.

입력형 계산 확인:

- 학생이 `k` 등 경계를 입력하면 그 구간의 확률만 표시한다. 학습지가 요구하는 판단·해석은 학생이 쓴다.

### 11-5. 정규분포·초과확률 (09 축제부스)

핵심 개념: 정규근사와 표준화, 특정 값을 넘을 확률(꼬리 면적).

입력값:

- 평균 `μ`, 표준편차 `σ`, 기준값 `x`(또는 이항의 `n`, `p`에서 정규근사)

핵심 계산:

```text
z = (x - μ) / σ
P(X > x) = 표준정규 꼬리 면적
```

핵심 그림·강조:

- 정규곡선과 기준값 오른쪽(또는 왼쪽) **면적을 색으로 강조**한다.
- `x`를 움직이면 면적(초과확률)이 함께 바뀐다. 강조 대상은 "초과 영역".
- 표준정규 값은 앱 내부 근사 함수로 계산하되(외부 CDN 없이), 학습지 제공 표와 어긋나지 않게 반올림 자리를 맞춘다.

입력형 계산 확인:

- 학생이 `x`(또는 `z`)를 입력하면 해당 확률만 표시한다.

### 11-6. 직렬·병렬 신뢰도 (07 직렬병렬)

핵심 개념: 부품 신뢰도의 결합. 직렬은 곱, 병렬은 여사건의 곱.

입력값:

- 각 부품 신뢰도 `r_i`, 구조(직렬/병렬/혼합)

핵심 계산:

```text
직렬: R = r1 × r2 × ... × rn
병렬: R = 1 - (1 - r1)(1 - r2)...(1 - rn)
혼합: 부분 구조를 위 두 규칙으로 축약
```

핵심 그림·강조:

- 부품 블록을 직렬/병렬 배치로 그리고, 구조를 바꾸면 전체 신뢰도 `R`이 변하는 것을 보여준다.
- 직렬은 약한 부품이 전체를 끌어내리는 점, 병렬은 중복이 신뢰도를 올리는 점을 강조한다.

입력형 계산 확인:

- 학생이 각 `r_i`와 구조를 입력하면 `R`만 계산한다.
