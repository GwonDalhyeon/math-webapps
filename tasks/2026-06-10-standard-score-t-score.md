# 표준점수와 T점수 변환 웹앱 및 학습지 작업지시서

웹앱 이름:
점수가 올랐으면 성적도 올랐을까?

부제:
표준편차 눈금으로 보는 표준점수와 T점수

위치:
`probability-statistics/standard-score-t-score/index.html`

학습지 위치:
`probability-statistics/standard-score-t-score/worksheet.md`

대상 학생:
고등학교 확률과 통계 과목에서 확률변수의 평균과 표준편차, `aX+b`의 평균과 표준편차, 표준점수, T점수를 배우는 일반계 고등학교 학생

수업 상황:
한 학생이 같은 과목을 여러 시기에 걸쳐 시험 보았다고 생각한다. 시험마다 난이도와 집단의 점수 분포가 달라질 수 있으며, 원점수는 `72 -> 78 -> 82`처럼 계속 오른다. 교사는 "점수가 올랐으면 정말 성적이 오른 것일까?"라는 질문으로 수업을 시작한다. 원점수가 실제로 올랐다는 사실과 집단 안에서의 상대적 위치가 올랐다는 판단은 구분해야 한다. 각 시험의 평균과 표준편차가 다르면 원점수만으로는 학생의 상대적 위치 변화를 판단하기 어렵다는 점을 수직선으로 보여준다.

핵심 질문:

```text
점수가 올랐으면, 정말 성적도 오른 것일까?
```

핵심 메시지:

```text
표준화는 어떤 점수가 평균에서 표준편차 몇 칸만큼 떨어져 있는지를 나타내는 과정이다.
T점수는 그 위치를 평균 50, 표준편차 10인 점수 체계로 다시 표현한 것이다.
```

---

## 학습 목표

- 원점수 상승이 항상 상대적 위치 상승을 뜻하지는 않음을 이해한다.
- 평균 `m`과 표준편차 `sigma`를 기준으로 수직선 눈금을 만들 수 있다.
- `X - m`은 평균을 0으로 옮기는 변환임을 이해한다.
- `(X - m) / sigma`는 표준편차를 1로 맞추는 변환임을 이해한다.
- `10(X - m) / sigma + 50`은 평균 50, 표준편차 10인 T점수 변환임을 이해한다.
- 변환 단계마다 평균과 표준편차가 어떻게 바뀌는지 계산할 수 있다.
- 일반적으로 `Y = a(X - m) / sigma + b`로 평균 `b`, 표준편차 `a`인 점수 체계로 변환할 수 있음을 이해한다.

## 기본 예시 데이터

웹앱의 기본 상태는 다음 예시를 사용한다.

| 시기 | 원점수 X | 시험 평균 m | 표준편차 sigma | 표준점수 Z | T점수 |
|---|---:|---:|---:|---:|---:|
| 1차 시험 | 72 | 60 | 12 | 1.0 | 60 |
| 2차 시험 | 78 | 72 | 6 | 1.0 | 60 |
| 3차 시험 | 82 | 80 | 4 | 0.5 | 55 |

의도:

- 원점수는 `72 -> 78 -> 82`로 계속 오른다.
- 그러나 표준점수는 `1.0 -> 1.0 -> 0.5`이다.
- 따라서 원점수는 실제로 올랐지만, 3차 시험에서의 상대적 위치는 1차와 2차보다 낮아졌다고 해석할 수 있다.

화면에 보여줄 최종 해석 문장:

```text
원점수만 보면 점수는 계속 올랐습니다.
하지만 3차 시험은 평균보다 표준편차 0.5개만큼 높은 위치입니다.
1차와 2차의 표준점수 1.0보다 낮으므로, 상대적 위치는 오히려 낮아졌습니다.
```

## 화면 구성 원칙

내용이 많아 보이지 않도록 화면 모드는 2개만 둔다.

```text
[점수 변화 보기] [표준화 과정 보기]
```

또는 탭 대신 위쪽에 같은 역할의 세그먼트 버튼을 둔다.

### 1. 점수 변화 보기

목표:
원점수 변화와 표준화된 위치 변화를 한 화면에서 비교한다.

구성:

1. 상단 질문 카드
   - "점수가 올랐으면 정말 성적도 오른 것일까?"
   - 짧은 설명 1문장만 둔다.

2. 입력 표
   - 1차, 2차, 3차 시험의 `X`, `m`, `sigma`를 교사가 수정할 수 있게 한다.
   - 기본값은 위의 예시 데이터를 사용한다.
   - `sigma`는 0보다 커야 한다.
   - 입력값을 바꾸면 모든 시각화와 계산 결과가 즉시 갱신된다.

3. 원점수 변화 수직선 또는 간단한 선 그래프
   - 가로축에 원점수를 표시한다.
   - 1차, 2차, 3차 점수를 연결해 원점수가 오르는 모습을 보여준다.
   - 이 그래프에서는 원점수가 실제로 올랐다는 사실만 확인한다.
   - 이 단계에서는 상대적 위치가 올랐는지에 대한 판단은 아직 보류한다.

4. 시험별 표준편차 눈금 수직선
   - 1차, 2차, 3차를 세 줄로 표시한다.
   - 각 줄은 해당 시험의 평균 `m`을 가운데에 두고 `m-2sigma`, `m-sigma`, `m`, `m+sigma`, `m+2sigma` 눈금을 표시한다.
   - 해당 시험의 원점수 `X`를 점 또는 핀으로 표시한다.
   - 점 옆에 `Z = (X-m)/sigma` 값을 표시한다.
   - 색만으로 구분하지 말고 `1차`, `2차`, `3차` 텍스트 라벨을 함께 표시한다.

5. 공통 T점수 수직선
   - 한 줄의 공통 수직선 위에 1차, 2차, 3차 T점수를 표시한다.
   - 기본 예시에서는 1차와 2차가 `T=60`에서 겹치고, 3차가 `T=55`에 놓인다.
   - 같은 위치에 여러 점이 겹치면 살짝 위아래로 벌리거나 작은 배지로 함께 표시한다.

6. 해석 박스
   - 화면 아래 또는 오른쪽에 짧은 결론을 표시한다.
   - 원점수 변화와 T점수 변화를 비교하는 문장으로 제한한다.

중요:
이 화면에서는 단계별 변환식 전체를 동시에 길게 펼치지 않는다. 학생이 먼저 "왜 표준화가 필요한가"를 보게 한다.

### 2. 표준화 과정 보기

목표:
선택한 한 시험에 대해 `X`가 T점수로 바뀌는 과정을 단계별로 보여준다.

구성:

1. 시험 선택 버튼
   - `1차 시험`, `2차 시험`, `3차 시험`
   - 기본 선택은 `1차 시험`

2. 단계 표시 버튼
   - `처음부터` : 1단계부터 다시 보기 (선택한 시험은 유지)
   - `다음 단계` : 한 단계씩 추가로 공개
   - `전체 보기` : 5단계 전체를 한꺼번에 공개

   `처음부터`와 별개의 `초기화` 버튼은 만들지 않는다. 동작이 동일하므로 버튼 하나로 충분하다.

3. 단계별 변환 표

| 단계 | 변수 | 현재 값 | 평균 | 표준편차 | 의미 |
|---|---|---:|---:|---:|---|
| 1 | `X` | `X` | `m` | `sigma` | 원래 점수 |
| 2 | `X - m` | `X-m` | `0` | `sigma` | 평균을 0으로 옮김 |
| 3 | `(X-m)/sigma` | `Z` | `0` | `1` | 표준편차를 1칸으로 맞춤 |
| 4 | `10(X-m)/sigma` | `10Z` | `0` | `10` | 표준편차를 10칸으로 늘림 |
| 5 | `10(X-m)/sigma + 50` | `T` | `50` | `10` | 평균을 50으로 옮김 |

4. 단계별 수직선
   - 선택한 단계까지만 수직선을 보여준다.
   - 각 단계의 평균 위치와 현재 값 위치를 표시한다.
   - 각 단계의 눈금 한 칸이 해당 단계의 표준편차임을 명확히 표시한다.

5. 계산 과정 표시
   - 선택한 시험의 실제 숫자로 계산을 보여준다.
   - 예: 1차 시험

```text
X - m = 72 - 60 = 12
(X - m) / sigma = 12 / 12 = 1
10 x 1 = 10
10 + 50 = 60
```

6. 일반화 박스
   - 표준화 과정 화면 맨 아래에 작게 둔다.
   - 처음부터 크게 강조하지 않는다.

```text
Z = (X - m) / sigma
Y = aZ + b = a(X - m) / sigma + b

Z의 평균은 0, 표준편차는 1이다.
Y의 평균은 b, 표준편차는 a이다. 단, 이 앱에서는 a > 0으로 둔다.
T점수는 a = 10, b = 50인 경우이다.
```

## 사용자 조작

필수 조작:

- 각 시험의 원점수 `X` 입력
- 각 시험의 평균 `m` 입력
- 각 시험의 표준편차 `sigma` 입력
- 예시 데이터로 되돌리는 `기본 예시` 버튼
- 표준화 과정 보기에서 시험 선택
- 표준화 과정 보기에서 `처음부터`, `다음 단계`, `전체 보기`

입력 처리 주의:

- 입력값이 바뀔 때 입력 테이블 DOM 전체를 재생성하지 않는다.
- Z, T 결과값 셀만 갱신하거나, `change` 이벤트(포커스 이탈 시 발생)를 사용한다.
- 숫자를 연속 입력할 수 있어야 한다. 한 자리 입력마다 포커스가 풀리지 않아야 한다.

선택 조작:

- 일반화 박스에서 목표 표준편차 `a`, 목표 평균 `b`를 바꿔 `Y = aZ + b`를 확인하는 작은 입력
- 기본값은 `a=10`, `b=50`
- 이 조작은 화면 하단 또는 접기 영역에 두어 본 수업 흐름을 방해하지 않게 한다.

사용하지 말 것:

- 과목 A, B, C처럼 여러 과목을 나열하는 비교
- 4개 이상의 시험 시기
- 정규분포 곡선의 면적 계산
- 등급, 백분위, 석차 계산
- 복잡한 애니메이션

## 시각화 세부 기준

### 원점수 변화 시각화

- 원점수 수직선 또는 간단한 가로 선 그래프를 사용한다.
- 기본 예시에서 `72 -> 78 -> 82`가 오른쪽으로 이동하는 모습이 보여야 한다.
- 이 시각화는 "원점수는 실제로 올랐다"는 도입 역할이다.
- 단, 상대적 위치가 올랐다는 결론은 이 시각화만으로 내리지 않는다.

### 시험별 표준편차 눈금 수직선

각 시험마다 다음 눈금을 표시한다.

```text
m-2sigma, m-sigma, m, m+sigma, m+2sigma
```

예: 1차 시험 `m=60`, `sigma=12`

```text
36, 48, 60, 72, 84
```

표시 원칙:

- 평균 `m`은 굵은 기준선 또는 강조 라벨로 표시한다.
- 원점수 `X`는 점 또는 핀으로 표시한다.
- `X`가 `m ± 2sigma` 범위를 벗어나면 수직선을 자동으로 확장하거나 가장자리에 화살표와 값을 표시한다.
- 눈금 간격은 화면에서 균등해야 한다. 실제 수치 간격이 아니라 "표준편차 한 칸"이라는 의미가 드러나야 한다.
- 점 라벨이 눈금 라벨과 겹치지 않도록 배치한다. 여러 점이 서로 가까울 때는 라벨을 위아래로 엇갈려 배치하되, 수직선 컨테이너 높이 안에서 벗어나지 않게 한다.

### 공통 T점수 수직선

- 기본 범위는 `30`부터 `70` 또는 데이터에 맞춘 자동 범위로 둔다.
- 평균 `50`을 가운데 기준선으로 표시한다.
- 표준편차 10 단위 눈금 `30, 40, 50, 60, 70`을 표시한다.
- 각 시험의 T점수를 같은 수직선 위에 올린다.

## 수학 모델 또는 계산식

표기 주의:
수업 화면에서는 학생이 보는 원점수를 간단히 `X`라고 표시한다. 엄밀히는 시험 점수 전체를 나타내는 확률변수를 `X`, 한 학생의 관찰 점수를 `x`로 구분할 수 있지만, 이 웹앱에서는 고등학생 가독성을 위해 화면 표시를 `X`로 통일한다.

각 시험 `i`에 대해:

```text
Z_i = (X_i - m_i) / sigma_i
T_i = 10Z_i + 50
```

단계별 평균과 표준편차:

```text
X의 평균 = m, 표준편차 = sigma
X - m의 평균 = 0, 표준편차 = sigma
(X - m) / sigma의 평균 = 0, 표준편차 = 1
10(X - m) / sigma의 평균 = 0, 표준편차 = 10
10(X - m) / sigma + 50의 평균 = 50, 표준편차 = 10
```

일반화:

```text
Z = (X - m) / sigma
Y = aZ + b = a(X - m) / sigma + b
Y의 평균 = b
Y의 표준편차 = a
```

주의:
고등학교 도입 수업에서는 `a > 0`으로 제한한다. 음수 `a`는 방향이 뒤집혀 표준편차가 `|a|`가 되는 별도 논의가 필요하므로 이 웹앱의 기본 흐름에서는 다루지 않는다.

### 2. 평균에서 얼마나 떨어졌을까?

### 4. 변환 과정 정리

다음 빈칸을 채운다.

```text
X - m은 평균을 ______로 옮기는 과정이다.
(X - m) / σ는 표준편차를 ______로 맞추는 과정이다.
T = 10(X - m) / σ + 50은 평균을 ______, 표준편차를 ______로 만드는 과정이다.
```

### 5. 일반화

간단한 확장 문항:

```text
Z = (X - m) / σ 라고 하자.
평균이 b, 표준편차가 a인 새 점수 Y로 바꾸려면

Y = ______ Z + ______

따라서 Y = ______________________________
```

정답 의도:

```text
Y = aZ + b
Y = a(X - m) / σ + b
```

학습지에서 피할 것:

- 너무 많은 시험 시기 추가
- 별도 과목 비교
- 정규분포 곡선 면적 문제
- 긴 개념 설명
- 계산칸이 부족한 과밀한 표

## 완료 기준

웹앱:

- 기본 예시에서 원점수 `72 -> 78 -> 82`가 상승하는 모습이 보인다.
- 기본 예시에서 T점수 `60, 60, 55`가 계산된다.
- 시험별 수직선에서 평균과 표준편차 눈금이 올바르게 표시된다.
- 공통 T점수 수직선에서 1차와 2차가 같은 위치, 3차가 더 낮은 위치에 표시된다.
- 선택한 시험의 표준화 과정이 `X`, `X-m`, `(X-m)/sigma`, `10(X-m)/sigma`, `10(X-m)/sigma+50` 순서로 드러난다.
- 각 단계의 평균과 표준편차가 표에 올바르게 표시된다.
- 입력값을 바꾸면 계산 결과와 시각화가 즉시 갱신된다.
- `sigma <= 0` 입력은 허용하지 않거나 오류 문구를 보여준다.
- `X`, `m`, `sigma` 입력 시 포커스가 유지된 채 연속 입력이 가능하다. 한 자리 입력마다 포커스가 풀리지 않아야 한다.
- 일반화 박스의 `b` 값이 0일 때도 올바르게 처리된다. `Number(value) || 0` 패턴은 b=0 입력을 잘못 처리할 수 있으므로, 명시적으로 처리한다.
- 자동 실행 타이머가 필요 없다. 만약 단계 애니메이션을 넣더라도 중복 타이머가 생기지 않게 한다.
- 모바일에서도 입력 표, 수직선, 해석 문장이 겹치지 않는다.

학습지:

- `worksheet.md` 하나로 완성된다.
- A4 세로 1페이지를 목표로 작성되어 있다.
- 최대 2페이지를 넘지 않을 분량이다.
- 웹앱 기본 예시와 같은 숫자를 사용한다.
- 원점수 변화, 표준점수 계산, T점수 계산, 해석, 일반화가 모두 포함된다.
- 학생이 직접 계산하고 한 문장으로 해석할 공간이 있다.

## 검증 방법

1. 파일 구조 확인

```text
probability-statistics/standard-score-t-score/index.html
probability-statistics/standard-score-t-score/worksheet.md
```

2. 링크 확인

- `probability-statistics/index.html`에서 새 카드가 실제 폴더로 연결되는지 확인한다.
- 존재하지 않는 파일로 가는 링크가 없는지 확인한다.

3. 브라우저 동작 확인

- 로컬에서 `probability-statistics/standard-score-t-score/index.html`을 연다.
- 브라우저 콘솔 오류가 없는지 확인한다.
- `X`, `m`, `sigma` 값을 바꿨을 때 Z점수와 T점수가 즉시 바뀌는지 확인한다.
- `점수 변화 보기`와 `표준화 과정 보기` 전환이 동작하는지 확인한다.
- `다음 단계`, `전체 보기`, `초기화` 버튼이 안정적으로 동작하는지 확인한다.

4. 수학 계산 확인

기본 예시:

```text
1차: (72 - 60) / 12 = 1, T = 60
2차: (78 - 72) / 6 = 1, T = 60
3차: (82 - 80) / 4 = 0.5, T = 55
```

5. 학습지 출력 확인

- `worksheet.md`를 Markdown 미리보기로 열고 A4 세로 인쇄 미리보기를 확인한다.
- 표가 잘리지 않는지 확인한다.
- 1페이지를 목표로 하되, 환경에 따라 2페이지까지는 허용한다.
