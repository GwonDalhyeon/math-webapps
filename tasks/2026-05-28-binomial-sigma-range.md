# 이항분포의 고정 범위 확률 웹앱 작업지시서

웹앱 이름:
이항분포의 고정 범위 확률

위치:
`probability-statistics/binomial-sigma-range/index.html`

대상 학생:
고등학교 확률과 통계 과목에서 이항분포, 기댓값, 표준편차를 배우는 일반계 고등학교 학생

수업 상황:
한 학생이 같은 유형의 문제를 `n`문제 푼다고 생각한다. 한 문제를 맞힐 확률이 `p`일 때, 맞힌 문제 수를 확률변수 `X`라고 하면 `X ~ B(n, p)`이다. 교사는 "범위 폭이 똑같아도 p가 다르면 확률이 달라질까?"라는 질문으로 수업을 시작한다.

학습 목표:
- 이항분포 `B(n, p)`에서 기댓값 `m = np`와 표준편차 `sigma = sqrt(np(1-p))`를 시각적으로 이해한다.
- 범위를 `m - a <= X <= m + a` (폭 2a, σ와 무관한 고정값)로 고정했을 때, p마다 확률이 다르게 나오는 이유를 σ 차이로 설명한다.
- `a`가 커지면 범위가 넓어져 확률이 증가하는 것을 확인한다.
- 비교 표의 `a/σ` 열을 통해 "σ가 크면 같은 범위가 분포를 덜 담는다"는 관계를 파악한다.
- `p = 1/2`에서 σ가 최대이므로 고정 범위 확률이 가장 낮고, p가 0 또는 1에 가까울수록 σ가 작아 확률이 높아진다는 것을 산술기하 부등식과 연결한다.

핵심 개념:
- 이항분포
- 기댓값
- 표준편차
- 고정 범위 확률 (범위 폭 2a 고정)
- a/σ 비율 (범위가 표준편차의 몇 배인지)
- 이산확률분포에서의 구간 확률
- 산술기하 부등식

---

## 설계 원칙: m ± a (고정 폭) vs m ± aσ

이 웹앱에서 범위는 `m - a <= X <= m + a`이다.

`m ± a*sigma`를 사용하면 연속 분포에서 `P(-a <= Z <= a)`와 같아지므로 p에 관계없이 확률이 거의 일정해진다. 이 경우 비교 표의 확률 열에 의미 있는 변화가 나타나지 않는다.

`m ± a` (고정 폭)를 사용하면 p마다 σ가 다르므로 같은 범위에서도 확률이 실제로 달라진다. p = 0.5에서 σ 최대 → a/σ 최소 → 확률 최소라는 반직관적이고 교육적으로 풍부한 결과가 나온다.

## 시각화 요소

### 이항분포 막대그래프

- 가로축: 성공 횟수 `k`
- 세로축: `P(X = k)`
- 범위 밖 막대: 연파랑(`#93c5fd`)
- 범위 안 막대 (`lo <= k <= hi`): 진파랑(`#1d4ed8`)
- 평균 `m = np`: 빨간 **실선**
- 연속 경계 `m - a`, `m + a`: 주황 **점선**
- 강조 배경: 정수 범위 구간에 연파랑 반투명 배경
- 범례: 범위 밖, 범위 안, 평균 m (실선), 경계 m±a (점선)
- 차트 하단 주석: "점선 경계(m ± a)는 실수값, 파란 막대는 그것을 올림/내림해 얻은 정수 범위"

> 평균선(실선)과 경계선(점선) 스타일 분리 방법:
> 1. `setLineDash([])` → 평균선 그리기
> 2. `setLineDash([5, 4])` → 경계선 그리기
> 3. `setLineDash([])` → 리셋

### p(1-p) 보조 그래프

- 가로축: `p` (0~1), 세로축: `p(1-p)` (0~0.25)
- 포물선 곡선 (`p(1-p)`)
- 현재 `p` 위치에 빨간 점 + 좌표 표시
- `p = 1/2` 위치에 주황 수직 점선 + 레이블
- 패널 제목: "p(1−p) — σ 크면 확률 낮아짐"
  (p(1-p) 최대 → σ 최대 → a/σ 최소 → 확률 최소라는 인과관계 요약)

### p별 실시간 비교 표

- 비교할 `p` 값: `0.1, 0.2, ..., 0.9`
- 열 구성:
  - `p`
  - 평균 `m`
  - 표준편차 `σ`
  - `a/σ` (범위가 σ의 몇 배인지 — 이 값이 클수록 확률 높음)
  - 정수 범위
  - 범위 안 확률
- `n` 또는 `a`가 바뀌면 표 전체 즉시 재계산
- 현재 슬라이더 `p`에 가장 가까운 행 **하나만** 강조 (`P_VALUES.reduce()`로 단일 최근접값 선택)

> a/σ 열이 핵심이다. p = 0.5에서 σ가 최대이므로 a/σ가 최소이고, 따라서 범위 안 확률도 가장 낮다.
> 학생은 이 열을 보면서 "a/σ가 크면 확률이 높고, 작으면 확률이 낮다"는 패턴을 직접 확인할 수 있다.

## 상태/결과 표시

현재 결과 패널:

- 분포: `X ~ B(n, p)`
- 평균 m = np
- 표준편차 σ = √(np(1−p))
- 범위 m ± a (점선): `m-a ~ m+a` (소수 둘째 자리 실수값)
- 실제 정수 범위 (올림/내림): `lo ≤ X ≤ hi`
  - `lo = max(0, ceil(m - a))`
  - `hi = min(n, floor(m + a))`
- 범위 안 확률 (백분율 소수 첫째 자리)

해석 문구 (interp-box):
- 범위 실수값 → 정수 범위 대응 표시
- `a/σ` 값: "범위가 σ의 X.XX배"
- `p(1-p)/0.25` 비율 기반 퍼짐 설명:
  - 비율 ≥ 0.90: "p가 1/2에 가까워 σ 최대 → 같은 범위에서 확률 최소"
  - 비율 ≥ 0.60: "최대의 XX% 수준"
  - 비율 < 0.60: "σ 작음 → 좁은 분포 → 같은 범위에서 확률 높음"

## 수학 모델 또는 계산식

- 확률변수: `X ~ B(n, p)`
- 확률질량함수: `P(X = k) = C(n, k) p^k (1-p)^(n-k)`
- 기댓값: `m = E(X) = np`
- 표준편차: `sigma = sqrt(np(1-p))`
- 범위: `m - a <= X <= m + a` (a는 고정 정수, σ와 무관)
- 실제 정수 범위:
  - `lower = max(0, ceil(m - a))`
  - `upper = min(n, floor(m + a))`
- 범위 안 확률: `sum P(X = k)` for `lower <= k <= upper`
- a/σ 비율: `a / sigma` (σ가 크면 작아짐 → 확률 낮아짐)
- 산술기하 부등식:
  - `p(1-p) <= 1/4`, 등호는 `p = 1/2`일 때
  - p = 1/2에서 σ 최대 → a/σ 최소 → 고정 범위 확률 최소

## 계산 구현 주의

- PMF 재귀 계산으로 오버플로 방지:
  ```
  P(X=0) = (1-p)^n
  P(X=k+1) = P(X=k) * ((n-k)/(k+1)) * (p/(1-p))
  ```
- `update()` 함수에서 `pmf = calcPMF(n, p)` 1회 계산 후 `updateResults`, `drawMain`에 인자로 전달한다.
- 비교 표는 9개 p값 각각의 PMF가 필요하므로 내부에서 별도 계산한다.
- 강조 행은 `P_VALUES.reduce()`로 단일 최근접값 선택 — threshold 방식 금지.
- σ ≈ 0인 경우(p 극단값) a/σ → ∞ 처리: `sigma > 0.01` 조건부로 분기.
- 경계선 점선: `setLineDash([])` 리셋을 평균선 전후, 경계선 후에 반드시 수행.

## 교사용 시연 흐름

1. 기본값 `n = 40`, `p = 0.5`, `a = 5`에서 시작한다.
2. 막대그래프에서 빨간 실선(평균)과 주황 점선(범위 경계 m±a)을 설명한다.
3. `a = 3`, `a = 5`, `a = 8`로 바꾸며 범위가 넓어질수록 확률이 커지는 것을 확인한다.
4. `a = 5`로 고정한 뒤 `p = 0.1, 0.3, 0.5, 0.7, 0.9`를 눌러 확률 변화를 관찰한다.
   - "p = 0.5일 때 확률이 가장 낮다"는 점을 강조한다.
5. 비교 표에서 `a/σ` 열이 p = 0.5에서 최소임을 확인한다.
   - "a/σ가 작을수록 확률이 낮다"는 패턴을 읽어낸다.
6. p(1-p) 보조 그래프에서 p = 0.5일 때 꼭짓점임을 보여준다.
7. 산술기하 부등식 박스에서 인과관계를 정리한다:
   - "p(1-p) 최대 → σ 최대 → a/σ 최소 → 확률 최소"

## 검증 방법

- 기본값 `n=40`, `p=0.5`, `a=5`에서:
  - 평균 20.00, σ ≈ 3.162, 범위 15.00~25.00, 정수 범위 15≤X≤25, a/σ ≈ 1.58 확인
- `p = 0.1`로 바꾸면:
  - 평균 4.00, σ ≈ 1.897, a/σ ≈ 2.64 → 확률이 p=0.5보다 높아야 함
- 비교 표에서 p=0.5 행의 확률이 가장 낮고 p=0.1, 0.9 행이 높은 U자 패턴 확인
- `p` 슬라이더를 0.15로 놓았을 때 강조 행이 하나만 표시되는지 확인
- `a` 버튼 3→5→8로 바꿀 때 확률이 단조증가하는지 확인
- 개발자 도구 콘솔 오류 없음 확인
- 화면 폭을 줄였을 때 패널 겹침 없음 확인
